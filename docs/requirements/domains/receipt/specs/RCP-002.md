## Summary

Defines the `Receipt` struct representing the result of executing a single transaction within a block. Contains the transaction identifier, execution status, fees, and state information.

## Specification

### Struct

```rust
pub struct Receipt {
    pub tx_id: Bytes32,
    pub block_height: u64,
    pub tx_index: u32,
    pub status: ReceiptStatus,
    pub fee_charged: u64,
    pub post_state_root: Bytes32,
    pub cumulative_fees: u64,
}
```

### Field Descriptions

| Field              | Type            | Description                                              |
|--------------------|-----------------|----------------------------------------------------------|
| `tx_id`            | `Bytes32`       | Hash of the transaction this receipt corresponds to.     |
| `block_height`     | `u64`           | Height of the block containing the transaction.          |
| `tx_index`         | `u32`           | Position of the transaction within the block.            |
| `status`           | `ReceiptStatus` | Outcome of the transaction execution.                    |
| `fee_charged`      | `u64`           | Fee deducted from the sender for this transaction.       |
| `post_state_root`  | `Bytes32`       | State root after applying this transaction.              |
| `cumulative_fees`  | `u64`           | Running total of fees up to and including this receipt.   |

## Acceptance Criteria

- [ ] `Receipt` struct contains all seven required fields with correct types.
- [ ] All fields are accessible for reading.
- [ ] `tx_id` is a `Bytes32` hash uniquely identifying the transaction.
- [ ] `status` uses the `ReceiptStatus` enum defined in RCP-001.
- [ ] `cumulative_fees` reflects the running total through this transaction's position.

## Implementation Notes

- `post_state_root` enables light client verification by providing an intermediate state commitment after each transaction.
- `cumulative_fees` is monotonically increasing within a block and must equal the sum of all `fee_charged` values up to and including this receipt.
- `tx_index` is zero-based within the block.
