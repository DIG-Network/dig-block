## Summary

Defines the aggregate query methods on `ReceiptList` for computing summary statistics: total count, success count, failure count, and total fees.

## Specification

### len

```rust
impl ReceiptList {
    pub fn len(&self) -> usize {
        self.receipts.len()
    }
}
```

Returns the number of receipts in the list.

### success_count

```rust
impl ReceiptList {
    pub fn success_count(&self) -> usize {
        self.receipts.iter().filter(|r| matches!(r.status, ReceiptStatus::Success)).count()
    }
}
```

Returns the count of receipts with `ReceiptStatus::Success`.

### failure_count

```rust
impl ReceiptList {
    pub fn failure_count(&self) -> usize {
        self.receipts.iter().filter(|r| !matches!(r.status, ReceiptStatus::Success)).count()
    }
}
```

Returns the count of receipts with any non-success status.

### total_fees

```rust
impl ReceiptList {
    pub fn total_fees(&self) -> u64 {
        self.receipts.iter().map(|r| r.fee_charged).sum()
    }
}
```

Returns the sum of `fee_charged` across all receipts.

## Acceptance Criteria

- [ ] `len()` returns the correct number of receipts.
- [ ] `len()` returns 0 for an empty list.
- [ ] `success_count()` counts only receipts with `ReceiptStatus::Success`.
- [ ] `failure_count()` counts all receipts with non-success status.
- [ ] `success_count() + failure_count() == len()` for any list.
- [ ] `total_fees()` returns the sum of all `fee_charged` values.
- [ ] `total_fees()` returns 0 for an empty list.

## Implementation Notes

- `failure_count` is defined as the complement of `success_count`. Any status other than `Success` is considered a failure for counting purposes.
- `total_fees` includes fees from both successful and failed transactions, since fees are charged regardless of execution outcome.
- These methods are used by the checkpoint builder (CKP-006) to aggregate statistics for the epoch.
