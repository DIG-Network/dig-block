# STV-003: Puzzle Hash Cross-Check

- **Normative:** [NORMATIVE.md](../NORMATIVE.md#stv-003-puzzle-hash-cross-check)
- **Verification:** [VERIFICATION.md](../VERIFICATION.md)
- **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
- **Spec:** SPEC Section 7.5.2

## Summary

For each removal, the puzzle hash recorded in the coin state database MUST match the puzzle hash declared in the CoinSpend. This cross-check ensures the spender is referencing the correct coin.

## Specification

```rust
fn check_puzzle_hashes(
    &self,
    exec: &ExecutionResult,
    coins: &dyn CoinLookup,
) -> Result<(), BlockError> {
    for spend_bundle in &self.body.spend_bundles {
        for coin_spend in &spend_bundle.coin_spends {
            let coin_id = coin_spend.coin.coin_id();

            // Only check persistent coins (ephemeral coins are not in the database)
            if let Some(coin_state) = coins.get_coin_state(&coin_id) {
                if coin_state.coin.puzzle_hash != coin_spend.coin.puzzle_hash {
                    return Err(BlockError::PuzzleHashMismatch {
                        coin_id,
                        expected: coin_state.coin.puzzle_hash,
                        declared: coin_spend.coin.puzzle_hash,
                    });
                }
            }
        }
    }

    Ok(())
}
```

## Acceptance Criteria

- [ ] For each removal, `coin_state.coin.puzzle_hash` is compared to `coin_spend.coin.puzzle_hash`
- [ ] Mismatch rejects with `BlockError::PuzzleHashMismatch`
- [ ] Uses `chia-protocol::CoinState.coin` field
- [ ] Ephemeral coins (not in database) are skipped for this check
- [ ] Chia parity with Check 20

## Implementation Notes

- This check is complementary to EXE-002 (which verifies puzzle_reveal hashes to puzzle_hash). This check verifies that the declared puzzle_hash matches what is in the state database.
- Together, EXE-002 and STV-003 form a complete chain: `tree_hash(puzzle_reveal) == coin_spend.puzzle_hash == coin_state.puzzle_hash`.
