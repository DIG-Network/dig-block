# STV-001: validate_state API

- **Normative:** [NORMATIVE.md](../NORMATIVE.md#stv-001-validate_state-api)
- **Verification:** [VERIFICATION.md](../VERIFICATION.md)
- **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
- **Spec:** SPEC Section 7.5

## Summary

L2Block MUST provide `validate_state` accepting an `ExecutionResult` from Tier 2, a `CoinLookup` trait for querying coin state, and the proposer's public key. Returns the computed state root on success. A convenience `validate_full()` method combines all three validation tiers.

## Specification

```rust
impl L2Block {
    pub fn validate_state(
        &self,
        exec: &ExecutionResult,
        coins: &dyn CoinLookup,
        proposer_pubkey: &PublicKey,
    ) -> Result<Bytes32, BlockError> {
        // STV-002: Coin existence checks for each removal
        self.check_coin_existence(exec, coins)?;

        // STV-003: Puzzle hash cross-check
        self.check_puzzle_hashes(exec, coins)?;

        // STV-004: Addition non-existence
        self.check_addition_uniqueness(exec, coins)?;

        // STV-005: Height/time lock evaluation
        self.evaluate_pending_assertions(exec, coins)?;

        // STV-006: Proposer signature verification
        self.verify_proposer_signature(proposer_pubkey)?;

        // STV-007: State root verification
        let computed_root = self.compute_and_verify_state_root(exec, coins)?;

        Ok(computed_root)
    }

    pub fn validate_full(
        &self,
        clvm_config: &ValidationConfig,
        genesis_challenge: &Bytes32,
        coins: &dyn CoinLookup,
        proposer_pubkey: &PublicKey,
    ) -> Result<Bytes32, BlockError> {
        // Tier 1: Structural validation
        self.validate_structure()?;

        // Tier 2: Execution validation
        let exec = self.validate_execution(clvm_config, genesis_challenge)?;

        // Tier 3: State validation
        self.validate_state(&exec, coins, proposer_pubkey)
    }
}
```

## Acceptance Criteria

- [ ] `validate_state` method exists on `L2Block` with the specified signature
- [ ] Accepts `&ExecutionResult` from Tier 2 execution validation
- [ ] Accepts `&dyn CoinLookup` for querying coin state
- [ ] Accepts `&PublicKey` for proposer signature verification
- [ ] Returns `Ok(Bytes32)` containing the computed state root on success
- [ ] Returns `Err(BlockError)` on any validation failure
- [ ] `validate_full()` combines all 3 tiers in order

## Implementation Notes

- The `CoinLookup` trait abstracts over the coin state database. It must support `get_coin_state(coin_id) -> Option<CoinState>` at minimum.
- `validate_full()` is a convenience wrapper. Each tier can also be called independently for partial validation or testing.
