## Summary

L2Block is the full block type containing the header, spend bundles, slash proposal payloads, and the proposer's signature. It provides convenience methods that delegate to the header.

## Specification

L2Block MUST contain the following fields:

- `header`: L2BlockHeader - the block header
- `spend_bundles`: Vec\<SpendBundle\> - the spend bundles included in this block
- `slash_proposal_payloads`: Vec\<Vec\<u8\>\> - raw slash proposal payloads
- `proposer_signature`: Signature - BLS signature from the block proposer

L2Block MUST provide the following methods:

- `new()` - constructor accepting all fields
- `hash()` - delegates to `header.hash()`, returning the block's identity hash
- `height()` - delegates to `header.height`, returning the block height
- `epoch()` - delegates to `header.epoch`, returning the block epoch

## Acceptance Criteria

- L2Block contains all four specified fields with correct types
- `new()` constructs a valid L2Block from all required fields
- `hash()` returns the same value as `header.hash()`
- `height()` returns `header.height`
- `epoch()` returns `header.epoch`
- SpendBundle is from chia-protocol
- Signature is from chia-bls

## Implementation Notes

- SpendBundle is imported from the chia-protocol crate
- Signature is imported from the chia-bls crate
- The block hash is defined entirely by the header; the body (spend bundles, slash proposals) is committed to via Merkle roots in the header
- slash_proposal_payloads are raw bytes to allow flexible encoding of slash evidence
