# BLK-001: L2BlockHeader Struct

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

L2BlockHeader is the core block header type containing all fields necessary to identify, anchor, and commit to the state of an L2 block. It groups fields into core identity, state commitments, L1 anchor, block metadata, L1 proof anchors, slash proposals, and DFSP roots.

## Specification

L2BlockHeader MUST contain the following field groups:

**Core identity:**
- `version`: u16 - protocol version
- `height`: u64 - block height in the L2 chain
- `epoch`: u64 - epoch number
- `parent_hash`: Bytes32 - hash of the parent block header

**State commitments:**
- `state_root`: Bytes32 - root of the state tree after applying this block
- `spends_root`: Bytes32 - Merkle root of spend bundles
- `additions_root`: Bytes32 - Merkle root of additions
- `removals_root`: Bytes32 - Merkle root of removals
- `receipts_root`: Bytes32 - Merkle root of receipts

**L1 anchor:**
- `l1_height`: u32 - Chia L1 block height this block is anchored to
- `l1_hash`: Bytes32 - Chia L1 block hash this block is anchored to

**Block metadata:**
- `timestamp`: u64 - block timestamp (seconds since Unix epoch)
- `proposer_index`: u32 - index of the block proposer
- `spend_bundle_count`: u32 - number of spend bundles in the block
- `total_cost`: Cost - total CLVM cost of all spends
- `total_fees`: u64 - total fees collected
- `additions_count`: u32 - number of coin additions
- `removals_count`: u32 - number of coin removals
- `block_size`: u32 - serialized block size in bytes
- `filter_hash`: Bytes32 - BIP158 compact block filter hash
- `extension_data`: Bytes32 - reserved for future extension

**L1 proof anchors:**
- 5 Optional\<Bytes32\> fields for L1 proof references

**Slash proposals:**
- `slash_proposal_count`: u32 - number of slash proposals in this block
- `slash_proposals_root`: Bytes32 - Merkle root of slash proposals

**DFSP roots:**
- 5 Bytes32 fields for Decentralized Fair Sequencing Protocol state roots

The struct MUST derive Serialize, Deserialize, Clone, Debug, PartialEq.

## Acceptance Criteria

- All field groups are present with correct types
- Struct derives Serialize, Deserialize, Clone, Debug, PartialEq
- All Bytes32 fields use the chia-protocol Bytes32 type
- Cost field uses the Cost type alias (u64)
- L1 proof anchor fields are Optional\<Bytes32\>
- Struct can be serialized to and deserialized from bytes

## Implementation Notes

- Bytes32 is re-exported from the chia-protocol crate
- Cost is a type alias for u64 (see BLK-006)
- The hash() method on L2BlockHeader should produce a SHA256 hash of the serialized header
- Optional fields for L1 proof anchors default to None
