# BLK-007: Version Auto-Detection

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 2.2](../../../../resources/SPEC.md)

## Summary

L2BlockHeader constructors automatically determine the protocol version from the block height, ensuring consistent versioning across the chain without requiring callers to manually track version transitions.

## Specification

L2BlockHeader constructors MUST auto-detect the protocol version using the following logic:

- If `height >= DFSP_ACTIVATION_HEIGHT`, then version = `VERSION_V2`
- If `height < DFSP_ACTIVATION_HEIGHT`, then version = `VERSION_V1`

When `DFSP_ACTIVATION_HEIGHT == u64::MAX` (the default), the condition `height >= u64::MAX` is only true when `height == u64::MAX`, which is not a practical block height. Therefore, in the default configuration, the version is always `VERSION_V1`.

Callers MUST NOT manually specify the version field. The version parameter MUST NOT be exposed in any constructor's public API.

## Acceptance Criteria

- Header constructed at height 0 has version == VERSION_V1
- Header constructed at height u64::MAX - 1 has version == VERSION_V1 (when DFSP_ACTIVATION_HEIGHT == u64::MAX)
- Header constructed at height == DFSP_ACTIVATION_HEIGHT has version == VERSION_V2 (when DFSP_ACTIVATION_HEIGHT < u64::MAX)
- Header constructed at height > DFSP_ACTIVATION_HEIGHT has version == VERSION_V2
- No constructor exposes a version parameter
- All constructors (new, new_with_collateral, new_with_l1_proofs, genesis) use the same auto-detection logic

## Implementation Notes

- The version detection logic should be a private helper function shared by all constructors
- Design Decision #10 in the spec mandates that version is derived, not specified, to prevent accidental version mismatches
- When DFSP is eventually activated, only DFSP_ACTIVATION_HEIGHT needs to change; no constructor signatures change
- The genesis block at height 0 will always be VERSION_V1 since DFSP_ACTIVATION_HEIGHT > 0 in any practical configuration
