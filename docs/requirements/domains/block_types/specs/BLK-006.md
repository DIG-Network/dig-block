> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)

## Summary

The crate defines fundamental type aliases and version constants, and re-exports key Chia ecosystem types to provide a unified type surface for DIG L2 block handling.

## Specification

The crate MUST define the following type aliases and constants:

**Type aliases:**
- `Cost` - type alias for u64, representing CLVM execution cost

**Version constants:**
- `VERSION_V1`: u16 = 1 - initial protocol version
- `VERSION_V2`: u16 = 2 - DFSP-enabled protocol version

**Re-exports:**
- `Bytes32` MUST be re-exported from the chia-protocol crate
- `Signature` MUST be re-exported from the chia-bls crate
- `PublicKey` MUST be re-exported from the chia-bls crate

## Acceptance Criteria

- `Cost` is a type alias for u64
- `VERSION_V1` == 1
- `VERSION_V2` == 2
- `Bytes32` is the same type as `chia_protocol::Bytes32`
- `Signature` is the same type as `chia_bls::Signature`
- `PublicKey` is the same type as `chia_bls::PublicKey`
- All types and constants are publicly accessible from the crate root

## Implementation Notes

- The `Cost` type alias provides semantic clarity: fields using `Cost` represent CLVM execution cost, distinguishing them from other u64 values like timestamps or fees
- Re-exporting Chia types avoids forcing downstream crates to depend directly on chia-protocol and chia-bls for common types
- VERSION_V1 is used for all blocks below DFSP_ACTIVATION_HEIGHT
- VERSION_V2 is used for blocks at or above DFSP_ACTIVATION_HEIGHT (see BLK-007)
