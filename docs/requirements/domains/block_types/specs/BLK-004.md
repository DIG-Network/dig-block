# BLK-004: L2Block Helper Methods

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

L2Block provides a comprehensive set of helper methods for computing Merkle roots, collecting coins, detecting integrity violations, and computing block size. These methods support block validation and state commitment verification.

## Specification

L2Block MUST provide the following helper methods:

**Merkle root computations:**
- `compute_spends_root()` - computes the Merkle root over all spend bundles in the block
- `compute_additions_root()` - computes the Merkle root over all additions, grouped by puzzle_hash for Chia parity
- `compute_removals_root()` - computes the Merkle root over all removals, maintaining Chia parity
- `compute_filter_hash()` - computes the BIP158 compact block filter hash
- `compute_slash_proposals_root()` - computes the Merkle root over all slash proposal payloads
- `slash_proposal_leaf_hash()` - computes the hash of a single slash proposal leaf

**Collection methods:**
- `all_additions() -> Vec<Coin>` - collects all addition coins across all spend bundles
- `all_addition_ids()` - collects all addition coin IDs across all spend bundles
- `all_removals() -> Vec<CoinId>` - collects all removal coin IDs across all spend bundles

**Integrity checks:**
- `has_duplicate_outputs() -> Option<Bytes32>` - detects duplicate output coins; returns the duplicate coin ID if found, None otherwise
- `has_double_spends() -> Option<Bytes32>` - detects double-spent input coins; returns the double-spent coin ID if found, None otherwise

**Size computation:**
- `compute_size() -> usize` - computes the serialized size of the full block in bytes

## Acceptance Criteria

- All Merkle root methods produce deterministic results for the same input
- `compute_additions_root()` groups additions by puzzle_hash for Chia parity
- `compute_filter_hash()` follows the BIP158 compact block filter specification
- `all_additions()` returns every coin created across all spend bundles
- `all_removals()` returns every coin spent across all spend bundles
- `has_duplicate_outputs()` returns Some(coin_id) when duplicates exist, None otherwise
- `has_double_spends()` returns Some(coin_id) when double spends exist, None otherwise
- `compute_size()` returns the byte count of the fully serialized block
- Empty blocks (no spend bundles) produce EMPTY_ROOT for spends_root

## Implementation Notes

- Merkle tree construction uses HASH_LEAF_PREFIX (0x01) and HASH_TREE_PREFIX (0x02) as defined in BLK-005
- Chia parity for additions_root means additions are sorted/grouped by puzzle_hash before tree construction
- BIP158 filter hash is used for light client support, allowing clients to check if a block might contain relevant transactions
- Coin type is from chia-protocol; CoinId is a Bytes32 identifying a specific coin
- The slash_proposal_leaf_hash method is a building block for compute_slash_proposals_root
