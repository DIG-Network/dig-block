# BLK-002: L2BlockHeader Constructors

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 2.2](../../../../resources/SPEC.md)

## Summary

L2BlockHeader provides multiple constructors for different use cases: a standard constructor with automatic version detection, a constructor with L1 collateral proof, a constructor with all L1 proof anchors, and a genesis block constructor.

## Specification

L2BlockHeader MUST provide the following constructors:

**`new()`** - Standard constructor that accepts all required fields and automatically detects the protocol version based on height (see BLK-007). Callers MUST NOT manually specify version.

**`new_with_collateral()`** - Constructor that accepts all standard fields plus an L1 collateral proof (Bytes32). Automatically detects version.

**`new_with_l1_proofs()`** - Constructor that accepts all standard fields plus all 5 L1 proof anchors. Automatically detects version.

**`genesis(network_id, l1_height, l1_hash)`** - Genesis block constructor that:
- Sets `parent_hash` to `network_id`
- Sets all count fields to zero (spend_bundle_count, additions_count, removals_count, slash_proposal_count)
- Sets all root fields to empty/zero values
- Sets height to 0
- Automatically detects version

## Acceptance Criteria

- `new()` returns a valid L2BlockHeader with version auto-detected from height
- `new_with_collateral()` includes the L1 collateral proof in the appropriate L1 proof anchor field
- `new_with_l1_proofs()` populates all 5 L1 proof anchor fields
- `genesis()` sets parent_hash to network_id, all counts to zero, all roots to empty
- No constructor allows the caller to manually specify the version field
- All constructors delegate version detection to the same logic (BLK-007)

## Implementation Notes

- Version auto-detection logic is shared across all constructors (see BLK-007)
- The genesis constructor is used to create the first block in a new L2 chain
- network_id serves as the parent_hash for the genesis block, linking the chain to its network identity
- Empty roots should use EMPTY_ROOT (see BLK-005), not ZERO_HASH

### Test Plan

| Test | Type | Description | Expected Result |
|------|------|-------------|-----------------|
| test_new_constructor | Unit | Call new() with valid fields | Returns header with auto-detected version |
| test_new_with_collateral | Unit | Call new_with_collateral() with collateral proof | Returns header with collateral proof in L1 proof anchor |
