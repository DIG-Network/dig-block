## Summary

The crate defines protocol-wide constants that govern block size limits, cost limits, hash prefixes, and sentinel values used throughout the DIG L2 protocol.

## Specification

The crate MUST define the following constants:

**Sentinel values:**
- `EMPTY_ROOT`: Bytes32 = SHA256("") = `0xe3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855`
- `ZERO_HASH`: Bytes32 = all zero bytes (32 bytes of 0x00)

**Block limits:**
- `MAX_BLOCK_SIZE`: u32 = 10_000_000 (10 MB)
- `MAX_COST_PER_BLOCK`: Cost = 550_000_000_000

**Slash proposal limits:**
- `MAX_SLASH_PROPOSALS_PER_BLOCK`: u32 = 64
- `MAX_SLASH_PROPOSAL_PAYLOAD_BYTES`: u32 = 65_536 (64 KB)

**Activation heights:**
- `DFSP_ACTIVATION_HEIGHT`: u64 = u64::MAX (effectively disabled until configured)

**Timestamp validation:**
- `MAX_FUTURE_TIMESTAMP_SECONDS`: u64 = 300 (5 minutes)

**Merkle tree hash prefixes:**
- `HASH_LEAF_PREFIX`: u8 = 0x01
- `HASH_TREE_PREFIX`: u8 = 0x02

## Acceptance Criteria

- EMPTY_ROOT equals SHA256 of the empty string
- ZERO_HASH is 32 bytes of zeros
- MAX_BLOCK_SIZE == 10_000_000
- MAX_COST_PER_BLOCK == 550_000_000_000
- MAX_SLASH_PROPOSALS_PER_BLOCK == 64
- MAX_SLASH_PROPOSAL_PAYLOAD_BYTES == 65_536
- DFSP_ACTIVATION_HEIGHT == u64::MAX
- MAX_FUTURE_TIMESTAMP_SECONDS == 300
- HASH_LEAF_PREFIX == 0x01
- HASH_TREE_PREFIX == 0x02
- All constants are publicly accessible from the crate

## Implementation Notes

- EMPTY_ROOT should be computed at compile time or as a lazy static to avoid runtime SHA256 computation
- ZERO_HASH can be defined as `Bytes32::default()` or `[0u8; 32]`
- DFSP_ACTIVATION_HEIGHT is set to u64::MAX to effectively disable DFSP until a governance decision activates it; at that point the constant would be updated
- HASH_LEAF_PREFIX and HASH_TREE_PREFIX follow the standard Merkle tree domain separation convention to prevent second-preimage attacks
