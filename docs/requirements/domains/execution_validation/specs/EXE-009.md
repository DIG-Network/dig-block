# EXE-009: PendingAssertion Type Definition

- **Normative:** [NORMATIVE.md](../NORMATIVE.md#exe-009-pendingassertion-type-definition)
- **Verification:** [VERIFICATION.md](../VERIFICATION.md)
- **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
- **Spec:** SPEC Sections 7.4.4, 7.4.7, 7.5.4

## Summary

`PendingAssertion` encapsulates a height or time lock assertion that is collected during Tier 2 (execution validation) and evaluated during Tier 3 (state validation). It captures the assertion kind, threshold value, and the coin ID needed for relative assertion lookups.

## Specification

```rust
use chia_protocol::Bytes32;

/// The kind of pending assertion, matching the 8 height/time lock condition opcodes.
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub enum AssertionKind {
    /// ASSERT_HEIGHT_ABSOLUTE(h): chain_height >= h
    HeightAbsolute(u64),
    /// ASSERT_HEIGHT_RELATIVE(h): chain_height >= confirmed_height + h
    HeightRelative(u64),
    /// ASSERT_SECONDS_ABSOLUTE(t): chain_timestamp >= t
    SecondsAbsolute(u64),
    /// ASSERT_SECONDS_RELATIVE(t): chain_timestamp >= coin_timestamp + t
    SecondsRelative(u64),
    /// ASSERT_BEFORE_HEIGHT_ABSOLUTE(h): chain_height < h
    BeforeHeightAbsolute(u64),
    /// ASSERT_BEFORE_HEIGHT_RELATIVE(h): chain_height < confirmed_height + h
    BeforeHeightRelative(u64),
    /// ASSERT_BEFORE_SECONDS_ABSOLUTE(t): chain_timestamp < t
    BeforeSecondsAbsolute(u64),
    /// ASSERT_BEFORE_SECONDS_RELATIVE(t): chain_timestamp < coin_timestamp + t
    BeforeSecondsRelative(u64),
}

/// A height/time lock assertion deferred from Tier 2 to Tier 3.
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize)]
pub struct PendingAssertion {
    /// The type and threshold value of the assertion.
    pub kind: AssertionKind,
    /// The coin ID that produced this assertion.
    /// Required for relative assertions to look up coin_confirmed_height / coin_timestamp.
    pub coin_id: Bytes32,
}

impl PendingAssertion {
    /// Construct a PendingAssertion from a chia-sdk-types Condition and the CoinSpend
    /// that produced it.
    pub fn from_condition(condition: &Condition, coin_spend: &CoinSpend) -> Option<Self> {
        let coin_id = coin_spend.coin.coin_id();
        let kind = match condition {
            Condition::AssertHeightAbsolute(a) => AssertionKind::HeightAbsolute(a.height as u64),
            Condition::AssertHeightRelative(a) => AssertionKind::HeightRelative(a.height as u64),
            Condition::AssertSecondsAbsolute(a) => AssertionKind::SecondsAbsolute(a.seconds),
            Condition::AssertSecondsRelative(a) => AssertionKind::SecondsRelative(a.seconds),
            Condition::AssertBeforeHeightAbsolute(a) => AssertionKind::BeforeHeightAbsolute(a.height as u64),
            Condition::AssertBeforeHeightRelative(a) => AssertionKind::BeforeHeightRelative(a.height as u64),
            Condition::AssertBeforeSecondsAbsolute(a) => AssertionKind::BeforeSecondsAbsolute(a.seconds),
            Condition::AssertBeforeSecondsRelative(a) => AssertionKind::BeforeSecondsRelative(a.seconds),
            _ => return None,
        };
        Some(Self { kind, coin_id })
    }
}
```

## Acceptance Criteria

- [ ] `AssertionKind` enum exists with 8 variants matching the 8 height/time lock condition opcodes
- [ ] `PendingAssertion` struct contains `kind: AssertionKind` and `coin_id: Bytes32`
- [ ] `PendingAssertion::from_condition()` correctly maps each height/time Condition variant
- [ ] `from_condition()` returns `None` for non-height/time conditions
- [ ] Both types derive `Debug`, `Clone`, `PartialEq`, `Eq`, `Serialize`, `Deserialize`
- [ ] `PendingAssertion` is used as the element type of `ExecutionResult.pending_assertions`
- [ ] `PendingAssertion` is consumed by `STV-005` height/time lock evaluation

## Implementation Notes

- `coin_id` is always populated because all 8 assertion types can be relative (relative assertions need the coin's confirmed height/timestamp for evaluation).
- The `from_condition()` factory method keeps the mapping between `chia-sdk-types::Condition` variants and `AssertionKind` in one place, avoiding scattered match arms.
- Relative assertions need the coin's `created_height` and creation timestamp from `CoinLookup` during Tier 3.
