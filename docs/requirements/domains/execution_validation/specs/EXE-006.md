# EXE-006: Coin Conservation and Fee Consistency

- **Normative:** [NORMATIVE.md](../NORMATIVE.md#exe-006-coin-conservation-and-fee-consistency)
- **Verification:** [VERIFICATION.md](../VERIFICATION.md)
- **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
- **Spec:** SPEC Section 7.4.6

## Summary

Per-bundle coin conservation is checked inside dig-clvm (total_input >= total_output). At the block level, computed_total_fees MUST equal header.total_fees. Reserve fee is checked by dig-clvm internally.

## Specification

```rust
// Per-bundle conservation: handled inside dig_clvm::validate_spend_bundle()
// dig-clvm verifies: sum(input_amounts) >= sum(output_amounts)
// dig-clvm verifies: fees >= reserve_fee (from RESERVE_FEE conditions)

// Block-level fee consistency: handled in dig-block
fn verify_fee_consistency(
    spend_results: &[SpendResult],
    header: &L2BlockHeader,
) -> Result<(), BlockError> {
    let computed_total_fees: u64 = spend_results
        .iter()
        .map(|r| r.fees)
        .sum();

    if computed_total_fees != header.total_fees {
        return Err(BlockError::FeesMismatch {
            computed: computed_total_fees,
            header: header.total_fees,
        });
    }

    Ok(())
}
```

## Acceptance Criteria

- [ ] Per-bundle conservation (`total_input >= total_output`) is enforced by dig-clvm
- [ ] Block-level `computed_total_fees` equals `header.total_fees`
- [ ] Mismatch rejects with `BlockError::FeesMismatch`
- [ ] Reserve fee checked by dig-clvm internally (propagates `ReserveFeeFailed`)
- [ ] Chia parity with Check 16 (conservation) and Check 19 (fees)

## Implementation Notes

- Fees are implicitly defined as `total_input - total_output` for each spend bundle. The block header declares the expected total fees, and the validator independently computes and compares.
- The reserve fee condition (`RESERVE_FEE`) is an assertion that the fee for a given spend meets a minimum threshold. This is checked within dig-clvm at the per-bundle level.
