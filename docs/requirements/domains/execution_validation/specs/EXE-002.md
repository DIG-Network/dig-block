# EXE-002: Puzzle Hash Verification

- **Normative:** [NORMATIVE.md](../NORMATIVE.md#exe-002-puzzle-hash-verification)
- **Verification:** [VERIFICATION.md](../VERIFICATION.md)
- **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
- **Spec:** SPEC Section 7.4.2

## Summary

For each `CoinSpend`, the tree hash of the puzzle reveal MUST equal the coin's puzzle hash. This ensures the puzzle program presented during spending is the same one committed to when the coin was created.

## Specification

```rust
use clvm_utils::tree_hash;

fn verify_puzzle_hash(coin_spend: &CoinSpend, allocator: &mut Allocator) -> Result<(), BlockError> {
    let puzzle_node = coin_spend.puzzle_reveal.to_node_ptr(allocator)?;
    let computed_hash = tree_hash(allocator, puzzle_node);

    if computed_hash != coin_spend.coin.puzzle_hash {
        return Err(BlockError::PuzzleHashMismatch {
            coin_id: coin_spend.coin.coin_id(),
            expected: coin_spend.coin.puzzle_hash,
            computed: computed_hash.into(),
        });
    }

    Ok(())
}
```

## Acceptance Criteria

- [ ] `tree_hash(puzzle_reveal)` is computed using `clvm-utils::tree_hash()`
- [ ] Computed hash is compared against `coin.puzzle_hash`
- [ ] Mismatch rejects with `BlockError::PuzzleHashMismatch`
- [ ] Verification is performed for every `CoinSpend` in every `SpendBundle`
- [ ] Chia parity with `block_body_validation.py` Check 20

## Implementation Notes

- This check may be performed inside `dig-clvm` as part of `validate_spend_bundle`. If so, dig-block relies on the error propagation from dig-clvm.
- `tree_hash` operates on the CLVM node tree representation and produces a SHA-256 based Merkle hash.
