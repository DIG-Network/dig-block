# EXE-005: BLS Aggregate Signature Verification

- **Normative:** [NORMATIVE.md](../NORMATIVE.md#exe-005-bls-aggregate-signature-verification)
- **Verification:** [VERIFICATION.md](../VERIFICATION.md)
- **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
- **Spec:** SPEC Section 7.4.5

## Summary

BLS signature verification MUST be handled inside `dig_clvm::validate_spend_bundle()`. dig-block MUST NOT perform separate signature verification. All AGG_SIG variants must be supported. An optional BlsCache may be used for performance.

## Specification

```rust
// BLS verification is fully delegated to dig-clvm.
// dig-block only handles the error propagation.

// Inside dig_clvm::validate_spend_bundle():
// - Collects all AGG_SIG_* conditions from CLVM output
// - Constructs the aggregate verification message for each variant:
//   AGG_SIG_PARENT, AGG_SIG_PUZZLE, AGG_SIG_AMOUNT,
//   AGG_SIG_PUZZLE_AMOUNT, AGG_SIG_PARENT_AMOUNT,
//   AGG_SIG_PARENT_PUZZLE, AGG_SIG_UNSAFE, AGG_SIG_ME
// - Verifies the aggregate BLS signature against all collected pairs
// - Optional BlsCache accelerates repeated verification

// dig-block error handling:
fn handle_signature_result(result: Result<SpendResult, ValidationError>) -> Result<SpendResult, BlockError> {
    result.map_err(|e| match e {
        ValidationError::SignatureFailed => BlockError::SignatureFailed,
        other => map_validation_error(other),
    })
}
```

## Acceptance Criteria

- [ ] BLS signature verification is performed inside `dig_clvm::validate_spend_bundle()`
- [ ] dig-block does NOT perform separate BLS signature verification
- [ ] All AGG_SIG variants are supported (AGG_SIG_PARENT, AGG_SIG_PUZZLE, AGG_SIG_AMOUNT, AGG_SIG_PUZZLE_AMOUNT, AGG_SIG_PARENT_AMOUNT, AGG_SIG_PARENT_PUZZLE, AGG_SIG_UNSAFE, AGG_SIG_ME)
- [ ] Optional `BlsCache` MAY be used for performance
- [ ] Invalid signature results in `BlockError::SignatureFailed`
- [ ] Chia parity with `block_body_validation.py` Check 22

## Implementation Notes

- BLS aggregate signature verification is computationally expensive. The `BlsCache` caches pairing results to avoid redundant computation during repeated validation of the same block or similar blocks.
- The AGG_SIG domain separation (different prefixes for each variant) is handled by dig-clvm following the Chia consensus specification.
