# EXE-007: Cost Consistency Verification

- **Normative:** [NORMATIVE.md](../NORMATIVE.md#exe-007-cost-consistency-verification)
- **Verification:** [VERIFICATION.md](../VERIFICATION.md)
- **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
- **Spec:** SPEC Section 7.4.6

## Summary

The computed total cost (sum of all `SpendResult.conditions.cost`) MUST equal `header.total_cost`. This ensures the block header accurately reflects the computational resources consumed during CLVM execution.

## Specification

```rust
fn verify_cost_consistency(
    spend_results: &[SpendResult],
    header: &L2BlockHeader,
) -> Result<(), BlockError> {
    let computed_total_cost: Cost = spend_results
        .iter()
        .map(|r| r.conditions.cost)
        .sum();

    if computed_total_cost != header.total_cost {
        return Err(BlockError::CostMismatch {
            computed: computed_total_cost,
            header: header.total_cost,
        });
    }

    Ok(())
}
```

## Acceptance Criteria

- [ ] `computed_total_cost` is the sum of all `SpendResult.conditions.cost` values
- [ ] `computed_total_cost` MUST equal `header.total_cost`
- [ ] Mismatch rejects with `BlockError::CostMismatch`
- [ ] Chia parity with Check 9

## Implementation Notes

- The `Cost` type is typically `u64`. Each spend bundle's cost reflects the CLVM execution cost (number of CLVM operations weighted by their cost factors) plus condition processing overhead.
- Cost is computed by dig-clvm during CLVM execution and returned as part of `SpendResult.conditions.cost`. dig-block only aggregates and verifies the total.
