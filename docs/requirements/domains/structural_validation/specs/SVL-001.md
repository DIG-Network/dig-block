## Summary

`L2BlockHeader::validate()` must compute the expected block version from the block height relative to `DFSP_ACTIVATION_HEIGHT` and reject any header whose version does not match.

## Specification

The version detection logic follows a simple height-based gate:

```rust
fn expected_version(height: u64, dfsp_activation_height: u64) -> u8 {
    if dfsp_activation_height == u64::MAX {
        VERSION_V1 // 1
    } else if height >= dfsp_activation_height {
        VERSION_V2 // 2
    } else {
        VERSION_V1 // 1
    }
}
```

Within `L2BlockHeader::validate()`:

```rust
let expected = expected_version(self.height, DFSP_ACTIVATION_HEIGHT);
if self.version != expected {
    return Err(BlockError::InvalidVersion {
        expected,
        actual: self.version,
    });
}
```

Key behaviors:

- When `DFSP_ACTIVATION_HEIGHT == u64::MAX`, DFSP is permanently disabled and the expected version is always `VERSION_V1` (1).
- When `height >= DFSP_ACTIVATION_HEIGHT`, the expected version is `VERSION_V2` (2).
- When `height < DFSP_ACTIVATION_HEIGHT`, the expected version is `VERSION_V1` (1).

## Acceptance Criteria

- [ ] `validate()` returns `Ok` for a V1 header at height below activation.
- [ ] `validate()` returns `Ok` for a V2 header at height equal to activation.
- [ ] `validate()` returns `Ok` for a V2 header at height above activation.
- [ ] `validate()` returns `Err(BlockError::InvalidVersion)` for a V2 header below activation.
- [ ] `validate()` returns `Err(BlockError::InvalidVersion)` for a V1 header at or above activation.
- [ ] When `DFSP_ACTIVATION_HEIGHT == u64::MAX`, `validate()` always expects V1 regardless of height.

## Implementation Notes

- The sentinel value `u64::MAX` for `DFSP_ACTIVATION_HEIGHT` disables DFSP permanently.
- This check is Step 1 of header validation and runs before any other checks.
