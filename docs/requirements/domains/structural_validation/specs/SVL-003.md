## Summary

`L2BlockHeader::validate()` must enforce upper bounds on both the total computational cost and the serialized block size declared in the header.

## Specification

Two independent limit checks are performed:

```rust
// Step 3: Cost limit
if self.total_cost > MAX_COST_PER_BLOCK {
    return Err(BlockError::CostExceeded {
        cost: self.total_cost,
        max: MAX_COST_PER_BLOCK,
    });
}

// Step 4: Size limit
if self.block_size > MAX_BLOCK_SIZE {
    return Err(BlockError::TooLarge {
        size: self.block_size,
        max: MAX_BLOCK_SIZE,
    });
}
```

- `MAX_COST_PER_BLOCK` is the protocol-defined maximum cost budget for a single block.
- `MAX_BLOCK_SIZE` is the protocol-defined maximum serialized block size in bytes.
- Both checks use strict greater-than comparisons (values exactly at the limit are accepted).

## Acceptance Criteria

- [x] `validate()` returns `Ok` when `total_cost == MAX_COST_PER_BLOCK` (boundary).
- [x] `validate()` returns `Err(BlockError::CostExceeded)` when `total_cost == MAX_COST_PER_BLOCK + 1`.
- [x] `validate()` returns `Ok` when `block_size == MAX_BLOCK_SIZE` (boundary).
- [x] `validate()` returns `Err(BlockError::TooLarge)` when `block_size == MAX_BLOCK_SIZE + 1`.
- [x] `validate()` returns `Ok` when both values are well below limits.
- [x] `validate()` returns `Err(BlockError::CostExceeded)` when cost exceeds but size is within limit.
- [x] `validate()` returns `Err(BlockError::TooLarge)` when size exceeds but cost is within limit (if cost check passes first).

## Implementation Notes

- The cost check (Step 3) runs before the size check (Step 4). If cost is exceeded, the size check is not reached.
- These are header-level checks on declared values; the actual block size is re-verified in `validate_structure()` (SVL-006).
