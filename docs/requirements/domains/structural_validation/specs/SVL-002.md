# SVL-002: Header DFSP Root Pre-Activation Check

## Summary

`L2BlockHeader::validate()` must verify that all five DFSP roots are set to `EMPTY_ROOT` when the block height is below `DFSP_ACTIVATION_HEIGHT`. This prevents DFSP data from being committed before the protocol activates it.

## Specification

The five DFSP root fields are:

1. `collateral_registry_root`
2. `cid_state_root`
3. `node_registry_root`
4. `namespace_update_root`
5. `dfsp_finalize_commitment_root`

Within `L2BlockHeader::validate()`:

```rust
if self.height < DFSP_ACTIVATION_HEIGHT {
    let dfsp_roots = [
        self.collateral_registry_root,
        self.cid_state_root,
        self.node_registry_root,
        self.namespace_update_root,
        self.dfsp_finalize_commitment_root,
    ];
    for root in &dfsp_roots {
        if *root != EMPTY_ROOT {
            return Err(BlockError::InvalidData(
                "DFSP root must be EMPTY_ROOT before activation".into(),
            ));
        }
    }
}
```

- `EMPTY_ROOT` is the canonical empty Merkle root (`Bytes32::default()` or protocol-defined constant).
- This check is Step 2 of header validation and runs immediately after the version check.
- After activation (`height >= DFSP_ACTIVATION_HEIGHT`), the roots MAY contain non-empty values.

## Acceptance Criteria

- [x] `validate()` returns `Ok` when all five roots are `EMPTY_ROOT` at pre-activation height.
- [x] `validate()` returns `Err(BlockError::InvalidData)` when `collateral_registry_root` is non-empty at pre-activation height.
- [x] `validate()` returns `Err(BlockError::InvalidData)` when `cid_state_root` is non-empty at pre-activation height.
- [x] `validate()` returns `Err(BlockError::InvalidData)` when `node_registry_root` is non-empty at pre-activation height.
- [x] `validate()` returns `Err(BlockError::InvalidData)` when `namespace_update_root` is non-empty at pre-activation height.
- [x] `validate()` returns `Err(BlockError::InvalidData)` when `dfsp_finalize_commitment_root` is non-empty at pre-activation height.
- [x] `validate()` returns `Ok` when DFSP roots are non-empty at or above activation height.

## Implementation Notes

- All five roots must be checked individually so that any single non-empty root triggers rejection.
- This check only applies when `height < DFSP_ACTIVATION_HEIGHT`; post-activation roots are validated elsewhere.
