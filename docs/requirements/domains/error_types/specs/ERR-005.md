# ERR-005: SignerBitmapError and ReceiptError Enums

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 4.3, 4.4](../../../../resources/SPEC.md)

## Summary

SignerBitmapError defines error variants for signer bitmap operations, which track which validators have signed a checkpoint. ReceiptError defines error variants for transaction receipt operations. Both are small, focused error enums for their respective subsystems.

## Specification

SignerBitmapError MUST define the following variants:

```rust
#[derive(Debug, Clone, thiserror::Error)]
pub enum SignerBitmapError {
    #[error("index out of bounds: index={index}, max={max}")]
    IndexOutOfBounds {
        index: u32,
        max: u32,
    },

    #[error("too many validators: {0}")]
    TooManyValidators(usize),

    #[error("invalid bitmap length: expected={expected}, got={got}")]
    InvalidLength {
        expected: usize,
        got: usize,
    },

    #[error("validator count mismatch: expected={expected}, got={got}")]
    ValidatorCountMismatch {
        expected: u32,
        got: u32,
    },
}
```

ReceiptError MUST define the following variants:

```rust
#[derive(Debug, Clone, thiserror::Error)]
pub enum ReceiptError {
    #[error("invalid receipt data: {0}")]
    InvalidData(String),

    #[error("receipt not found: {0}")]
    NotFound(Bytes32),
}
```

Both enums MUST derive thiserror::Error to provide Display and std::error::Error implementations.

## Acceptance Criteria

- [ ] SignerBitmapError defines all 4 variants with correct names and fields
- [ ] IndexOutOfBounds carries index and max (both u32)
- [ ] TooManyValidators carries a usize count
- [ ] InvalidLength carries expected and got (both usize)
- [ ] ValidatorCountMismatch carries expected and got (both u32)
- [ ] ReceiptError defines both variants with correct names and fields
- [ ] ReceiptError::InvalidData carries a String description
- [ ] ReceiptError::NotFound carries a Bytes32 hash
- [ ] Both enums derive thiserror::Error with Display and Error impls

## Implementation Notes

- SignerBitmapError is used by the signer bitmap module which manages a bit vector tracking validator signatures
- IndexOutOfBounds is returned when attempting to set/get a bit beyond the bitmap capacity
- TooManyValidators is returned during bitmap creation if the validator set exceeds the maximum supported size
- InvalidLength is returned during deserialization when the bitmap byte length doesn't match the expected size for the validator count
- ValidatorCountMismatch is returned when a bitmap is used with a different validator count than it was created for
- ReceiptError is used by the receipt subsystem for transaction execution receipts
- ReceiptError::NotFound is used when looking up a receipt by its hash
