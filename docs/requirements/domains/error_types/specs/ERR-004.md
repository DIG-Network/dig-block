# ERR-004: BuilderError Enum

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 6.5](../../../../resources/SPEC.md)

## Summary

BuilderError defines error variants for the block builder, which incrementally assembles a block by adding spend bundles and slash proposals. These errors enforce budget limits and builder preconditions during block construction.

## Specification

BuilderError MUST define the following variants:

```rust
#[derive(Debug, Clone, thiserror::Error)]
pub enum BuilderError {
    #[error("cost budget exceeded: current={current}, addition={addition}, max={max}")]
    CostBudgetExceeded {
        current: u64,
        addition: u64,
        max: u64,
    },

    #[error("size budget exceeded: current={current}, addition={addition}, max={max}")]
    SizeBudgetExceeded {
        current: u32,
        addition: u32,
        max: u32,
    },

    #[error("too many slash proposals: max={max}")]
    TooManySlashProposals {
        max: u32,
    },

    #[error("slash proposal too large: size={size}, max={max}")]
    SlashProposalTooLarge {
        size: u32,
        max: u32,
    },

    #[error("signing failed: {0}")]
    SigningFailed(String),

    #[error("empty block: no spend bundles added")]
    EmptyBlock,

    #[error("missing DFSP roots")]
    MissingDfspRoots,
}
```

BuilderError MUST derive thiserror::Error to provide Display and std::error::Error implementations.

## Acceptance Criteria

- [ ] All 7 variants are defined with correct names
- [ ] CostBudgetExceeded carries current, addition, and max (all u64)
- [ ] SizeBudgetExceeded carries current, addition, and max (all u32)
- [ ] TooManySlashProposals carries max (u32)
- [ ] SlashProposalTooLarge carries size and max (both u32)
- [ ] SigningFailed carries a String reason
- [ ] EmptyBlock is a unit variant
- [ ] MissingDfspRoots is a unit variant
- [ ] thiserror::Error is derived, providing Display and Error impls

## Implementation Notes

- The builder tracks cumulative cost and size as spend bundles are added
- CostBudgetExceeded is returned when adding a spend bundle would exceed MAX_COST_PER_BLOCK
- SizeBudgetExceeded is returned when adding a spend bundle would exceed MAX_BLOCK_SIZE
- TooManySlashProposals is returned when the count would exceed MAX_SLASH_PROPOSALS_PER_BLOCK
- SlashProposalTooLarge is returned when a single proposal exceeds MAX_SLASH_PROPOSAL_PAYLOAD_BYTES
- EmptyBlock prevents producing a block with no transactions
- MissingDfspRoots is returned when version >= VERSION_V2 but DFSP roots were not supplied
