# ERR-003: CheckpointError Enum

## Summary

CheckpointError defines error variants for checkpoint creation, validation, and finalization. These errors cover the full lifecycle of a checkpoint from initial submission through finalization.

## Specification

CheckpointError MUST define the following variants:

```rust
#[derive(Debug, Clone, thiserror::Error)]
pub enum CheckpointError {
    #[error("invalid checkpoint data: {0}")]
    InvalidData(String),

    #[error("checkpoint not found for epoch {0}")]
    NotFound(u64),

    #[error("invalid checkpoint: {0}")]
    Invalid(String),

    #[error("score not higher: current={current}, submitted={submitted}")]
    ScoreNotHigher {
        current: u64,
        submitted: u64,
    },

    #[error("epoch mismatch: expected={expected}, got={got}")]
    EpochMismatch {
        expected: u64,
        got: u64,
    },

    #[error("checkpoint already finalized")]
    AlreadyFinalized,

    #[error("checkpoint process not started")]
    NotStarted,
}
```

CheckpointError MUST derive thiserror::Error to provide Display and std::error::Error implementations.

## Acceptance Criteria

- [ ] All 7 variants are defined with correct names
- [ ] InvalidData carries a String description
- [ ] NotFound carries a u64 epoch number
- [ ] Invalid carries a String description
- [ ] ScoreNotHigher carries current and submitted u64 values
- [ ] EpochMismatch carries expected and got u64 values
- [ ] AlreadyFinalized is a unit variant
- [ ] NotStarted is a unit variant
- [ ] thiserror::Error is derived, providing Display and Error impls

## Implementation Notes

- CheckpointError is separate from BlockError because checkpoints have a distinct lifecycle
- ScoreNotHigher prevents checkpoint regression -- a new checkpoint must have a higher score
- EpochMismatch catches checkpoint submissions for the wrong epoch
- AlreadyFinalized prevents modifications to a checkpoint that has been committed to L1
- NotStarted indicates an attempt to finalize or query a checkpoint before the process begins
