# ERR-002: BlockError Execution and State Variants

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 4.1, 7.6](../../../../resources/SPEC.md)

## Summary

BlockError also defines Tier 2 (execution validation) and Tier 3 (state validation) error variants. Tier 2 errors arise during CLVM execution and spend bundle verification. Tier 3 errors arise when checking block results against the state database.

## Specification

BlockError MUST define the following Tier 2 (execution) variants:

```rust
    // --- Tier 2: Execution validation ---

    #[error("puzzle hash mismatch for coin {coin_id}: expected={expected}, computed={computed}")]
    PuzzleHashMismatch {
        coin_id: Bytes32,
        expected: Bytes32,
        computed: Bytes32,
    },

    #[error("CLVM execution failed for coin {coin_id}: {reason}")]
    ClvmExecutionFailed {
        coin_id: Bytes32,
        reason: String,
    },

    #[error("CLVM cost exceeded for coin {coin_id}: cost={cost}, remaining={remaining}")]
    ClvmCostExceeded {
        coin_id: Bytes32,
        cost: u64,
        remaining: u64,
    },

    #[error("assertion failed: condition={condition}, reason={reason}")]
    AssertionFailed {
        condition: String,
        reason: String,
    },

    #[error("announcement not found: {announcement_hash}")]
    AnnouncementNotFound {
        announcement_hash: Bytes32,
    },

    #[error("signature verification failed for bundle index {bundle_index}")]
    SignatureFailed {
        bundle_index: u32,
    },

    #[error("coin minting: removed={removed}, added={added}")]
    CoinMinting {
        removed: u64,
        added: u64,
    },

    #[error("fees mismatch: header={header}, computed={computed}")]
    FeesMismatch {
        header: u64,
        computed: u64,
    },

    #[error("reserve fee failed: required={required}, actual={actual}")]
    ReserveFeeFailed {
        required: u64,
        actual: u64,
    },

    #[error("cost mismatch: header={header}, computed={computed}")]
    CostMismatch {
        header: u64,
        computed: u64,
    },
```

And the following Tier 3 (state) variants:

```rust
    // --- Tier 3: State validation ---

    #[error("invalid proposer signature")]
    InvalidProposerSignature,

    #[error("block not found: {0}")]
    NotFound(Bytes32),

    #[error("invalid state root: expected={expected}, computed={computed}")]
    InvalidStateRoot {
        expected: Bytes32,
        computed: Bytes32,
    },

    #[error("coin not found: {coin_id}")]
    CoinNotFound {
        coin_id: Bytes32,
    },

    #[error("coin already spent: {coin_id} at height {spent_height}")]
    CoinAlreadySpent {
        coin_id: Bytes32,
        spent_height: u64,
    },

    #[error("coin already exists: {coin_id}")]
    CoinAlreadyExists {
        coin_id: Bytes32,
    },
```

All variants MUST be part of the same BlockError enum (alongside Tier 1 from ERR-001).

## Acceptance Criteria

- [ ] All 10 Tier 2 variants are defined with correct names and fields
- [ ] All 6 Tier 3 variants are defined with correct names and fields
- [ ] PuzzleHashMismatch carries coin_id, expected, and computed
- [ ] ClvmExecutionFailed carries coin_id and reason (String)
- [ ] ClvmCostExceeded carries coin_id, cost, and remaining
- [ ] AssertionFailed carries condition and reason (both String)
- [ ] AnnouncementNotFound carries announcement_hash
- [ ] SignatureFailed carries bundle_index
- [ ] CoinMinting carries removed and added amounts
- [ ] FeesMismatch, ReserveFeeFailed, CostMismatch carry expected/actual pairs
- [ ] InvalidStateRoot carries expected and computed
- [ ] CoinNotFound carries coin_id
- [ ] CoinAlreadySpent carries coin_id and spent_height
- [ ] CoinAlreadyExists carries coin_id
- [ ] All variants produce meaningful Display output via thiserror

## Implementation Notes

- Tier 2 errors require CLVM execution -- they are more expensive to detect than Tier 1
- Tier 3 errors require state database access -- they are checked last
- The validation pipeline checks Tier 1 first, then Tier 2, then Tier 3, failing fast at each tier
- AssertionFailed covers all CLVM assertion conditions (ASSERT_SECONDS_RELATIVE, ASSERT_HEIGHT_ABSOLUTE, etc.)
- CoinMinting detects the case where a spend bundle creates more value than it destroys
- NotFound(Bytes32) is for referencing a block by hash that doesn't exist in the chain
