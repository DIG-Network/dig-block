## Summary

Defines the methods on `CheckpointSubmission` for querying submission state, checking attestation thresholds, and recording L1 submissions.

## Specification

### hash

```rust
impl CheckpointSubmission {
    pub fn hash(&self) -> Bytes32 {
        self.checkpoint.hash()
    }
}
```

Delegates to the underlying checkpoint's hash.

### epoch

```rust
impl CheckpointSubmission {
    pub fn epoch(&self) -> u64 {
        self.checkpoint.epoch
    }
}
```

Delegates to the underlying checkpoint's epoch.

### signing_percentage

```rust
impl CheckpointSubmission {
    pub fn signing_percentage(&self) -> u64 {
        self.signer_bitmap.signing_percentage()
    }
}
```

Returns the signing percentage from the signer bitmap.

### meets_threshold

```rust
impl CheckpointSubmission {
    pub fn meets_threshold(&self, threshold_pct: u64) -> bool {
        self.signer_bitmap.has_threshold(threshold_pct)
    }
}
```

Returns whether the signing percentage meets or exceeds the given threshold.

### record_submission

```rust
impl CheckpointSubmission {
    pub fn record_submission(&mut self, height: u32, coin_id: Bytes32) {
        self.submission_height = Some(height);
        self.submission_coin = Some(coin_id);
    }
}
```

Records the L1 block height and coin ID where the checkpoint was submitted.

### is_submitted

```rust
impl CheckpointSubmission {
    pub fn is_submitted(&self) -> bool {
        self.submission_height.is_some()
    }
}
```

Returns `true` if the submission has been recorded on L1.

## Acceptance Criteria

- [ ] `hash()` returns the same value as `checkpoint.hash()`.
- [ ] `epoch()` returns the same value as `checkpoint.epoch`.
- [ ] `signing_percentage()` delegates to `signer_bitmap.signing_percentage()`.
- [ ] `meets_threshold(threshold_pct)` returns correct boolean based on signing percentage.
- [ ] `record_submission(height, coin_id)` sets both `submission_height` and `submission_coin`.
- [ ] `is_submitted()` returns `false` before `record_submission` is called.
- [ ] `is_submitted()` returns `true` after `record_submission` is called.

## Implementation Notes

- `hash()` and `epoch()` are convenience accessors to avoid deep field access through the nested checkpoint.
- `record_submission` is called once the checkpoint has been confirmed on L1, recording the proof of inclusion.
- `is_submitted` provides a quick check without inspecting the `Option` fields directly.
