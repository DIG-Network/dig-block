## Summary

Defines the score computation method on `Checkpoint`. The score is used to determine the winning checkpoint submission for an epoch, combining stake participation with block coverage.

## Specification

### compute_score

```rust
impl Checkpoint {
    pub fn compute_score(&self, stake_percentage: u64) -> u64 {
        stake_percentage * self.block_count as u64
    }
}
```

Returns the product of `stake_percentage` and `block_count`. Higher scores indicate checkpoints with greater stake participation covering more blocks.

## Acceptance Criteria

- [ ] `compute_score(stake_percentage)` returns `stake_percentage * block_count`.
- [ ] Returns 0 when `stake_percentage` is 0.
- [ ] Returns 0 when `block_count` is 0.
- [ ] Handles large values without overflow within `u64` range.

## Implementation Notes

- `stake_percentage` is an integer 0-100 representing the submitter's share of total stake.
- The score favors checkpoints that cover more blocks with higher stake backing.
- In practice, `block_count` will be bounded by the epoch length (e.g., ~225 blocks per epoch), so overflow is not a concern for realistic inputs.
