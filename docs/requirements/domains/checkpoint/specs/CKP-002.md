## Summary

Defines the `CheckpointSubmission` struct that wraps a `Checkpoint` with attestation data and L1 submission tracking. The constructor creates a submission with no L1 submission recorded.

## Specification

### Struct

```rust
pub struct CheckpointSubmission {
    pub checkpoint: Checkpoint,
    pub signer_bitmap: SignerBitmap,
    pub aggregate_signature: Signature,
    pub aggregate_pubkey: PublicKey,
    pub score: u64,
    pub submitter: u32,
    pub submission_height: Option<u32>,
    pub submission_coin: Option<Bytes32>,
}
```

### Constructor

```rust
impl CheckpointSubmission {
    pub fn new(
        checkpoint: Checkpoint,
        signer_bitmap: SignerBitmap,
        aggregate_signature: Signature,
        aggregate_pubkey: PublicKey,
        score: u64,
        submitter: u32,
    ) -> Self {
        Self {
            checkpoint,
            signer_bitmap,
            aggregate_signature,
            aggregate_pubkey,
            score,
            submitter,
            submission_height: None,
            submission_coin: None,
        }
    }
}
```

## Acceptance Criteria

- [ ] `CheckpointSubmission` struct contains all eight required fields with correct types.
- [ ] `new()` sets `submission_height` to `None`.
- [ ] `new()` sets `submission_coin` to `None`.
- [ ] `new()` stores all provided parameters correctly.

## Implementation Notes

- `submission_height` and `submission_coin` are populated later via `record_submission()` (see CKP-005) when the checkpoint is submitted to L1.
- `submitter` is the validator index of the submitter.
- `score` is pre-computed via `Checkpoint::compute_score()` (see CKP-004).
