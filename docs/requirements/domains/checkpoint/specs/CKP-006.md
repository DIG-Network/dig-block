## Summary

Defines the `CheckpointBuilder` for incrementally constructing a `Checkpoint`. The builder accumulates blocks and withdrawals, then computes Merkle roots on `build()`.

## Specification

### Builder State

```rust
pub struct CheckpointBuilder {
    epoch: u64,
    prev_checkpoint: Bytes32,
    state_root: Bytes32,
    block_hashes: Vec<Bytes32>,
    tx_count: u64,
    total_fees: u64,
    withdrawal_hashes: Vec<Bytes32>,
}
```

### new

```rust
impl CheckpointBuilder {
    pub fn new(epoch: u64, prev_checkpoint: Bytes32) -> Self {
        Self {
            epoch,
            prev_checkpoint,
            state_root: Bytes32::default(),
            block_hashes: Vec::new(),
            tx_count: 0,
            total_fees: 0,
            withdrawal_hashes: Vec::new(),
        }
    }
}
```

Creates a new builder for the given epoch, linked to the previous checkpoint.

### add_block

```rust
impl CheckpointBuilder {
    pub fn add_block(&mut self, block_hash: Bytes32, tx_count: u64, fees: u64) {
        self.block_hashes.push(block_hash);
        self.tx_count += tx_count;
        self.total_fees += fees;
    }
}
```

Adds a block's hash and aggregates its transaction count and fees.

### set_state_root

```rust
impl CheckpointBuilder {
    pub fn set_state_root(&mut self, state_root: Bytes32) {
        self.state_root = state_root;
    }
}
```

Sets the final state root for the epoch.

### add_withdrawal

```rust
impl CheckpointBuilder {
    pub fn add_withdrawal(&mut self, withdrawal_hash: Bytes32) {
        self.withdrawal_hashes.push(withdrawal_hash);
    }
}
```

Adds a withdrawal hash to the checkpoint.

### build

```rust
impl CheckpointBuilder {
    pub fn build(self) -> Checkpoint {
        let block_root = merkle_root(&self.block_hashes);
        let withdrawals_root = merkle_root(&self.withdrawal_hashes);
        Checkpoint {
            epoch: self.epoch,
            state_root: self.state_root,
            block_root,
            block_count: self.block_hashes.len() as u32,
            tx_count: self.tx_count,
            total_fees: self.total_fees,
            prev_checkpoint: self.prev_checkpoint,
            withdrawals_root,
            withdrawal_count: self.withdrawal_hashes.len() as u32,
        }
    }
}
```

Builds the final checkpoint. Computes `block_root` as the Merkle root over accumulated block hashes and `withdrawals_root` as the Merkle root over accumulated withdrawal hashes.

## Acceptance Criteria

- [ ] `new(epoch, prev_checkpoint)` creates a builder with empty block and withdrawal lists.
- [ ] `add_block(block_hash, tx_count, fees)` accumulates blocks, tx counts, and fees.
- [ ] `set_state_root(state_root)` stores the state root.
- [ ] `add_withdrawal(withdrawal_hash)` accumulates withdrawal hashes.
- [ ] `build()` returns a `Checkpoint` with correct `block_count` and `withdrawal_count`.
- [ ] `build()` computes `block_root` as the Merkle root of block hashes.
- [ ] `build()` computes `withdrawals_root` as the Merkle root of withdrawal hashes.
- [ ] `build()` correctly aggregates `tx_count` and `total_fees` across all added blocks.
- [ ] Empty block list produces a valid default Merkle root.
- [ ] Empty withdrawal list produces a valid default Merkle root.

## Implementation Notes

- The Merkle root computation uses the same algorithm as other Merkle trees in the system.
- An empty list of hashes should produce `EMPTY_ROOT` (all zeros or a defined constant) for consistency.
- `build()` consumes the builder (takes ownership of `self`), preventing reuse.
- The builder pattern allows blocks to be added in order as they are produced during the epoch.
