## Summary

Defines the `Checkpoint` struct that represents an epoch summary anchored to L1. Contains aggregate statistics about blocks and transactions within the epoch, along with Merkle roots for verification.

## Specification

### Struct

```rust
pub struct Checkpoint {
    pub epoch: u64,
    pub state_root: Bytes32,
    pub block_root: Bytes32,
    pub block_count: u32,
    pub tx_count: u64,
    pub total_fees: u64,
    pub prev_checkpoint: Bytes32,
    pub withdrawals_root: Bytes32,
    pub withdrawal_count: u32,
}
```

### Constructor

```rust
impl Checkpoint {
    pub fn new() -> Self {
        Self {
            epoch: 0,
            state_root: Bytes32::default(),
            block_root: Bytes32::default(),
            block_count: 0,
            tx_count: 0,
            total_fees: 0,
            prev_checkpoint: Bytes32::default(),
            withdrawals_root: Bytes32::default(),
            withdrawal_count: 0,
        }
    }
}
```

## Acceptance Criteria

- [ ] `Checkpoint` struct contains all nine required fields with correct types.
- [ ] `new()` creates a default checkpoint with epoch 0, zero counts, and default hashes.
- [ ] All fields are accessible for reading and writing.

## Implementation Notes

- `block_root` is a Merkle root computed over all block hashes in the epoch.
- `withdrawals_root` is a Merkle root computed over all withdrawal hashes in the epoch.
- `prev_checkpoint` links checkpoints into a chain for verification continuity.
