# HSH-002: Checkpoint Hash

## Summary

Checkpoint hash is the canonical identifier for a checkpoint record. It is computed as SHA-256 over 9 fields in a fixed order totaling 160 bytes, with numeric fields in little-endian byte order.

## Specification

Checkpoint hash MUST be computed as follows:

```rust
pub fn hash(&self) -> Bytes32 {
    let mut hasher = Sha256::new();

    hasher.update(&self.epoch.to_le_bytes());             // 8 bytes
    hasher.update(self.state_root.as_ref());              // 32 bytes
    hasher.update(self.block_root.as_ref());              // 32 bytes
    hasher.update(&self.block_count.to_le_bytes());       // 8 bytes
    hasher.update(&self.tx_count.to_le_bytes());          // 8 bytes
    hasher.update(&self.total_fees.to_le_bytes());        // 8 bytes
    hasher.update(self.prev_checkpoint.as_ref());         // 32 bytes
    hasher.update(self.withdrawals_root.as_ref());        // 32 bytes
    hasher.update(&self.withdrawal_count.to_le_bytes());  // 8 bytes
    // Total: 160 bytes

    Bytes32::new(hasher.finalize())
}
```

Key rules:
- The field order MUST be exactly: epoch, state_root, block_root, block_count, tx_count, total_fees, prev_checkpoint, withdrawals_root, withdrawal_count
- Numeric fields (u64) MUST use little-endian encoding
- MUST use chia-sha2::Sha256

## Acceptance Criteria

- [ ] hash() produces SHA-256 over exactly 160 bytes
- [ ] Field order matches: epoch, state_root, block_root, block_count, tx_count, total_fees, prev_checkpoint, withdrawals_root, withdrawal_count
- [ ] Numeric fields are little-endian encoded
- [ ] Uses chia-sha2::Sha256
- [ ] Deterministic: same input always produces same output

## Implementation Notes

- The 160-byte total comes from: (4 * 8) + (4 * 32) + 8 = 32 + 128 = 160
- prev_checkpoint is ZERO_HASH for the first checkpoint in the chain
- Checkpoint hashes form a chain via prev_checkpoint, enabling efficient checkpoint verification
