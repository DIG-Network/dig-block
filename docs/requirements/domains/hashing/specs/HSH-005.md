# HSH-005: Removals Root Construction

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 3.5](../../../../resources/SPEC.md)

## Summary

The removals_root commits to all coins spent (removed) by a block. It is computed via chia-consensus::compute_merkle_set_root() over all removed coin IDs, matching Chia's block_body_validation.py:185. This ensures L2 removal Merkle proofs are compatible with Chia L1 tooling.

## Specification

removals_root MUST be computed as follows:

```rust
pub fn compute_removals_root(removals: &[Bytes32]) -> Bytes32 {
    if removals.is_empty() {
        return EMPTY_ROOT;
    }

    compute_merkle_set_root(removals)
}
```

Key rules:
- MUST use chia-consensus::compute_merkle_set_root() for Merkle set construction
- Input MUST be the list of all removed coin IDs (Bytes32)
- MUST match Chia's block_body_validation.py:185
- MUST return EMPTY_ROOT when no removals are present

## Acceptance Criteria

- [ ] Returns EMPTY_ROOT for empty removals list
- [ ] Uses chia-consensus::compute_merkle_set_root()
- [ ] Input is all removed coin IDs as Bytes32
- [ ] Output matches Chia's block_body_validation.py:185 for identical inputs
- [ ] Order-independent: same set of coin IDs produces the same root regardless of input order

## Implementation Notes

- Unlike additions_root (HSH-004), removals are not grouped -- each coin ID is a direct Merkle set item
- compute_merkle_set_root produces a Merkle set root which is order-independent
- This enables light clients to prove a coin was spent in a block
- Coin IDs are Bytes32 values derived from the coin's parent_coin_info, puzzle_hash, and amount
