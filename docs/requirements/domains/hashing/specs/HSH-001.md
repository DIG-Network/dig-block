# HSH-001: Block Header Hash

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 3.1](../../../../resources/SPEC.md)

## Summary

Block header hash is the canonical identifier for an L2 block. It is computed as SHA-256 over the fixed-order concatenation of all 33 header fields (626 bytes total), with numeric fields in little-endian byte order and optional fields hashing as ZERO_HASH when None.

## Specification

Block header hash MUST be computed as follows:

```rust
pub fn hash(&self) -> Bytes32 {
    let mut hasher = Sha256::new();

    // All 33 fields in SPEC Section 3.1 order:
    hasher.update(&self.version.to_le_bytes());           // 2 bytes
    hasher.update(&self.height.to_le_bytes());            // 8 bytes
    hasher.update(&self.epoch.to_le_bytes());             // 8 bytes
    hasher.update(self.parent_hash.as_ref());             // 32 bytes
    hasher.update(self.state_root.as_ref());              // 32 bytes
    hasher.update(self.spends_root.as_ref());             // 32 bytes
    hasher.update(self.additions_root.as_ref());          // 32 bytes
    hasher.update(self.removals_root.as_ref());           // 32 bytes
    hasher.update(self.receipts_root.as_ref());           // 32 bytes
    hasher.update(&self.l1_height.to_le_bytes());         // 4 bytes
    hasher.update(self.l1_hash.as_ref());                 // 32 bytes
    hasher.update(&self.timestamp.to_le_bytes());         // 8 bytes
    hasher.update(&self.proposer_index.to_le_bytes());    // 4 bytes
    hasher.update(&self.spend_bundle_count.to_le_bytes());// 4 bytes
    hasher.update(&self.total_cost.to_le_bytes());        // 8 bytes
    hasher.update(&self.total_fees.to_le_bytes());        // 8 bytes
    hasher.update(&self.additions_count.to_le_bytes());   // 4 bytes
    hasher.update(&self.removals_count.to_le_bytes());    // 4 bytes
    hasher.update(&self.block_size.to_le_bytes());        // 4 bytes
    hasher.update(self.filter_hash.as_ref());             // 32 bytes
    hasher.update(self.extension_data.as_ref());          // 32 bytes
    // 5 Optional L1 proof anchors: hash as ZERO_HASH when None
    // ... each 32 bytes
    hasher.update(&self.slash_proposal_count.to_le_bytes()); // 4 bytes
    hasher.update(self.slash_proposals_root.as_ref());    // 32 bytes
    // 5 DFSP roots: each 32 bytes
    // Total: 626 bytes

    Bytes32::new(hasher.finalize())
}
```

Key rules:
- The field order MUST exactly match SPEC Section 3.1
- Numeric fields (u16, u32, u64) MUST use little-endian encoding
- Optional\<Bytes32\> fields MUST hash as ZERO_HASH when None
- MUST use chia-sha2::Sha256 as the hashing primitive
- Given identical field values, hash() MUST always produce the same Bytes32

## Acceptance Criteria

- [ ] hash() produces SHA-256 over exactly 626 bytes
- [ ] Field order matches SPEC Section 3.1 (33 fields)
- [ ] Numeric fields are little-endian encoded
- [ ] Optional fields hash as ZERO_HASH when None
- [ ] Uses chia-sha2::Sha256
- [ ] Deterministic: same input always produces same output
- [ ] Changing any single field produces a different hash

## Implementation Notes

- The 626-byte total comes from: 2 + 8 + 8 + (21 * 32) + 4 + 4 + 4 + 8 + 8 + 4 + 4 + 4 + 4 = 626
- ZERO_HASH is 32 bytes of zeros (see BLK-005)
- chia-sha2::Sha256 is the same SHA-256 used throughout the Chia ecosystem
- This hash serves as the block's canonical identifier and is referenced by child blocks via parent_hash
