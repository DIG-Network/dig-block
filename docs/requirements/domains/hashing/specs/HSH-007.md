# HSH-007: Tagged Merkle Hashing

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 3.3](../../../../resources/SPEC.md)

## Summary

All block-level Merkle trees use tagged hashing to prevent second-preimage attacks. Leaf nodes are prefixed with 0x01 and internal nodes with 0x02 before hashing. These tag constants are built into chia-consensus::merkle_set and MUST NOT be redefined by dig-block. This matches Chia's merkle_utils.py.

## Specification

Tagged Merkle hashing MUST follow these rules:

```rust
// Leaf hash: SHA-256(0x01 || data)
fn hash_leaf(data: &[u8]) -> Bytes32 {
    let mut hasher = Sha256::new();
    hasher.update(&[HASH_LEAF_PREFIX]);  // 0x01
    hasher.update(data);
    Bytes32::new(hasher.finalize())
}

// Internal node hash: SHA-256(0x02 || left || right)
fn hash_node(left: &Bytes32, right: &Bytes32) -> Bytes32 {
    let mut hasher = Sha256::new();
    hasher.update(&[HASH_TREE_PREFIX]);  // 0x02
    hasher.update(left.as_ref());
    hasher.update(right.as_ref());
    Bytes32::new(hasher.finalize())
}
```

Key rules:
- Leaf hash MUST be SHA-256(0x01 || data)
- Internal node hash MUST be SHA-256(0x02 || left || right)
- These tag constants (HASH_LEAF_PREFIX = 0x01, HASH_TREE_PREFIX = 0x02) are built into chia-consensus::merkle_set
- dig-block MUST NOT redefine these constants for Merkle tree computation
- This prevents second-preimage attacks by making leaf and node hashes distinguishable
- Chia parity: merkle_utils.py uses the same tagging scheme

## Acceptance Criteria

- [ ] Leaf hash equals SHA-256(0x01 || data)
- [ ] Internal node hash equals SHA-256(0x02 || left || right)
- [ ] chia-consensus::merkle_set uses HASH_LEAF_PREFIX = 0x01 and HASH_TREE_PREFIX = 0x02
- [ ] dig-block does NOT redefine these constants for Merkle computation
- [ ] Tagged hashing matches Chia's merkle_utils.py behavior
- [ ] A leaf hash is never equal to a node hash for any input (domain separation)

## Implementation Notes

- HASH_LEAF_PREFIX (0x01) and HASH_TREE_PREFIX (0x02) are defined as protocol constants in BLK-005
- These constants are used by chia-consensus::merkle_set internally -- dig-block relies on the library's implementation
- The tagging prevents an attacker from constructing a value that hashes the same as a leaf when placed in a node position (or vice versa)
- dig-block may reference these constants for documentation and testing, but MUST NOT use independently-defined copies for actual Merkle tree computation
- All Merkle trees in the block (spends_root, additions_root, removals_root, slash_proposals_root) use this tagging
