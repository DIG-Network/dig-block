# HSH-003: Spends Root Computation

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 3.3](../../../../resources/SPEC.md)

## Summary

The spends_root commits to the ordered list of spend bundles in a block. It is computed as a Merkle tree root over the SHA-256 hash of each serialized SpendBundle, preserving block order. When no spend bundles are present, it returns EMPTY_ROOT.

## Specification

spends_root MUST be computed as follows:

```rust
pub fn compute_spends_root(spend_bundles: &[SpendBundle]) -> Bytes32 {
    if spend_bundles.is_empty() {
        return EMPTY_ROOT;
    }

    let leaves: Vec<Bytes32> = spend_bundles
        .iter()
        .map(|bundle| sha256(bundle.to_bytes()))
        .collect();

    MerkleTree::new(&leaves).root()
}
```

Key rules:
- MUST use chia-sdk-types::MerkleTree for Merkle tree construction
- Each leaf MUST be sha256(spend_bundle.to_bytes()) -- the SHA-256 of the serialized SpendBundle
- Leaves MUST be in block order (the order spend bundles appear in the block)
- MUST return EMPTY_ROOT (SHA-256 of empty string) when no spend bundles are present

## Acceptance Criteria

- [ ] Returns EMPTY_ROOT for an empty spend bundle list
- [ ] Computes Merkle root using chia-sdk-types::MerkleTree
- [ ] Each leaf is sha256 of the serialized SpendBundle
- [ ] Leaf order matches block order of spend bundles
- [ ] Single spend bundle produces sha256(sha256(bundle)) as root (Merkle tree of one leaf)
- [ ] Multiple spend bundles produce correct Merkle root

## Implementation Notes

- EMPTY_ROOT is defined in BLK-005 as SHA-256 of the empty string
- chia-sdk-types::MerkleTree handles the tagged hashing internally (see HSH-007)
- Block order is the canonical ordering -- reordering spend bundles would produce a different root
- This is used during structural validation to verify the header's spends_root field
