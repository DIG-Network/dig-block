# HSH-008: Receipts Root Computation

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 3.3](../../../../resources/SPEC.md)

## Summary

`receipts_root` MUST be computed via `chia-sdk-types::MerkleTree` over SHA-256 hashes of each `Receipt` in block order. This is the same construction used for `spends_root` and `slash_proposals_root`. Returns `EMPTY_ROOT` when the receipt list is empty.

## Specification

```rust
use chia_sdk_types::MerkleTree;
use chia_sha2::Sha256;

/// Compute the Merkle root over an ordered list of receipts.
///
/// Each receipt is hashed individually via SHA-256 over its bincode-serialized bytes,
/// then the hashes are fed into a MerkleTree (balanced binary Merkle tree with
/// tagged hashing: 0x01 leaf prefix, 0x02 node prefix).
///
/// Returns EMPTY_ROOT if receipts is empty.
pub fn compute_receipts_root(receipts: &[Receipt]) -> Bytes32 {
    if receipts.is_empty() {
        return EMPTY_ROOT;
    }

    let hashes: Vec<Bytes32> = receipts
        .iter()
        .map(|r| {
            let bytes = bincode::serialize(r).expect("receipt serialization should not fail");
            let mut hasher = Sha256::new();
            hasher.update(&bytes);
            Bytes32::new(hasher.finalize())
        })
        .collect();

    MerkleTree::new(&hashes).root()
}
```

The `receipts_root` field in `L2BlockHeader` is set by:
- `BlockBuilder::build()` — takes `receipts_root` as a parameter (caller computes it from `ReceiptList::root`)
- `ReceiptList::finalize()` — calls `compute_receipts_root()` internally
- `ReceiptList::from_receipts()` — calls `compute_receipts_root()` during construction

## Acceptance Criteria

- [ ] `compute_receipts_root()` uses `chia-sdk-types::MerkleTree` for tree construction
- [ ] Each receipt is hashed via SHA-256 of its bincode serialization
- [ ] Hashes are passed to `MerkleTree::new()` in block order (preserving receipt ordering)
- [ ] Returns `EMPTY_ROOT` when given an empty slice
- [ ] Tagged hashing (0x01 leaf, 0x02 node) is inherited from MerkleTree implementation
- [ ] Result matches `ReceiptList.root` after `finalize()`
- [ ] Used by `ReceiptList::from_receipts()` and `ReceiptList::finalize()`

## Implementation Notes

- The hashing of individual receipts uses bincode serialization followed by SHA-256, matching the pattern used for SpendBundle hashing in HSH-003.
- `MerkleTree` from `chia-sdk-types` automatically applies tagged hashing (HSH-007).
- The `compute_receipts_root()` function SHOULD be defined in `src/hash.rs` alongside other root computation functions.
