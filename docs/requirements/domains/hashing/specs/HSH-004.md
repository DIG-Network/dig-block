# HSH-004: Additions Root Construction

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 3.4](../../../../resources/SPEC.md)

## Summary

The additions_root commits to all coins created by a block. It is computed via chia-consensus::compute_merkle_set_root() over additions grouped by puzzle_hash, matching Chia's block_body_validation.py:158-175 exactly. This ensures L2 addition Merkle proofs are compatible with Chia L1 tooling.

## Specification

additions_root MUST be computed as follows:

```rust
pub fn compute_additions_root(additions: &[Coin]) -> Bytes32 {
    if additions.is_empty() {
        return EMPTY_ROOT;
    }

    // Group additions by puzzle_hash
    let mut by_puzzle_hash: HashMap<Bytes32, Vec<Bytes32>> = HashMap::new();
    for coin in additions {
        by_puzzle_hash
            .entry(coin.puzzle_hash)
            .or_default()
            .push(coin.coin_id());
    }

    // Build Merkle set items: [puzzle_hash, hash_coin_ids(coin_ids)]
    let mut items: Vec<Bytes32> = Vec::new();
    for (puzzle_hash, coin_ids) in &by_puzzle_hash {
        items.push(*puzzle_hash);
        items.push(hash_coin_ids(coin_ids));
    }

    compute_merkle_set_root(&items)
}
```

Key rules:
- MUST use chia-consensus::compute_merkle_set_root() for Merkle set construction
- Additions MUST be grouped by puzzle_hash
- For each puzzle_hash, items MUST be \[puzzle_hash, hash_coin_ids(coin_ids)\]
- MUST match Chia's block_body_validation.py:158-175 exactly
- MUST return EMPTY_ROOT when no additions are present

## Acceptance Criteria

- [ ] Returns EMPTY_ROOT for empty additions list
- [ ] Groups additions by puzzle_hash correctly
- [ ] For each puzzle_hash group, produces [puzzle_hash, hash_coin_ids(coin_ids)]
- [ ] Uses chia-consensus::compute_merkle_set_root()
- [ ] Output matches Chia's block_body_validation.py:158-175 for identical inputs
- [ ] Multiple coins with the same puzzle_hash are grouped together

## Implementation Notes

- hash_coin_ids() hashes the sorted list of coin IDs for a given puzzle_hash
- The grouping-by-puzzle_hash approach matches Chia L1's additions Merkle proof structure
- This enables light clients to prove a coin was created in a block using the same proof format as Chia L1
- compute_merkle_set_root uses a Merkle set (unordered) rather than a Merkle tree (ordered)
