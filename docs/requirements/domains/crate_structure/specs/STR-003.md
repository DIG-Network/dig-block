# STR-003: Public Re-exports in lib.rs

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 10](../../../../resources/SPEC.md)

## Summary
The crate's public API surface MUST be accessible directly from the crate root via re-exports in `lib.rs`. Consumers of dig-block should be able to import all primary types, errors, traits, and builders with a single `use dig_block::*` or targeted imports like `use dig_block::L2Block`, without navigating internal module paths.

## Specification
`lib.rs` MUST publicly re-export the following items:

**Block types:**
```rust
pub use types::header::L2BlockHeader;
pub use types::block::L2Block;
pub use types::attested::AttestedBlock;
pub use types::checkpoint::{Checkpoint, CheckpointSubmission};
```

**Status and supporting types:**
```rust
pub use types::status::{BlockStatus, CheckpointStatus};
pub use types::receipt::{Receipt, ReceiptList, ReceiptStatus};
pub use types::signer_bitmap::SignerBitmap;
```

**Error types:**
```rust
pub use error::{BlockError, CheckpointError, BuilderError, SignerBitmapError, ReceiptError};
```

**Constants:**
```rust
pub use constants::*;
```

**Traits:**
```rust
pub use traits::{CoinLookup, BlockSigner};
```

**Builder types:**
```rust
pub use builder::block_builder::BlockBuilder;
pub use builder::checkpoint_builder::CheckpointBuilder;
```

**Validation result:**
```rust
pub use validation::execution::ExecutionResult;
```

All re-exports MUST use `pub use` to ensure they are part of the crate's public API. The internal module structure (e.g., `types::header::L2BlockHeader`) SHOULD remain accessible but the flattened re-exports at crate root are the primary public interface.

## Implementation Notes
- Primary file: `src/lib.rs`
- Dependencies: All re-exported types must be defined and public in their respective modules
- Edge cases: If a type is renamed during development, the re-export must be updated to match. Glob re-exports (`pub use constants::*`) should be used sparingly and only where all items in the module are intended to be public.
