# STR-001: Cargo.toml Dependencies

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)

## Summary
The dig-block crate maximally reuses existing Chia ecosystem crates rather than reimplementing cryptographic primitives, consensus types, or serialization logic. Cargo.toml MUST declare all required dependencies at their specified minimum versions to ensure compatibility across the DIG Network stack.

## Specification
Cargo.toml MUST include the following dependencies:

```toml
[dependencies]
chia-protocol = "0.26"
chia-bls = "0.26"
dig-clvm = "0.1"
chia-consensus = "0.26"
chia-sdk-types = "0.30"
chia-sdk-signer = "0.30"
chia-sha2 = "0.26"
chia-traits = "0.26"
clvm-utils = "0.26"
clvmr = "0.14"
bincode = "*"
serde = { version = "*", features = ["derive"] }
thiserror = "*"
```

The Chia ecosystem crates provide:
- **chia-protocol**: Core protocol types including `Bytes32`, `Coin`, `CoinSpend`, `SpendBundle`, `CoinState`
- **chia-bls**: BLS12-381 signature types (`PublicKey`, `Signature`) and aggregation
- **dig-clvm**: DIG-specific CLVM puzzle logic and curry/uncurry utilities
- **chia-consensus**: Consensus validation rules (coin ID computation, signature verification)
- **chia-sdk-types**: Extended SDK types (conditions, puzzle representations)
- **chia-sdk-signer**: Spend signing utilities
- **chia-sha2**: SHA-256 hashing (`hash_256`)
- **chia-traits**: Streamable trait for canonical serialization
- **clvm-utils**: CLVM tree hashing and serialization helpers
- **clvmr**: CLVM runtime (Allocator, NodePtr) for puzzle execution
- **bincode**: Binary serialization for block wire format
- **serde**: Serialization framework with derive macros
- **thiserror**: Ergonomic error type derivation

## Acceptance Criteria
- [ ] Cargo.toml contains all 13 required dependencies
- [ ] chia-protocol, chia-bls, chia-consensus, chia-sha2, chia-traits, clvm-utils are at version 0.26 or higher
- [ ] chia-sdk-types and chia-sdk-signer are at version 0.30 or higher
- [ ] clvmr is at version 0.14 or higher
- [ ] dig-clvm is at version 0.1 or higher
- [ ] serde includes the "derive" feature
- [ ] `cargo check` succeeds with all dependencies resolved

## Implementation Notes
- Primary file: `Cargo.toml`
- Dependencies: All listed crates must be published and resolvable
- Edge cases: Version ranges should use minimum version specifiers; exact pinning is acceptable but not required for non-Chia crates (bincode, serde, thiserror)
