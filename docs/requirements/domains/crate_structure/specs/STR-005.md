# STR-005: Test Infrastructure

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 12](../../../../resources/SPEC.md)

## Summary
The dig-block crate MUST provide reusable test infrastructure including mock implementations of its core traits and helper functions for constructing test fixtures. This infrastructure enables thorough testing of validation, building, and serialization logic without requiring a live Chia full node or real BLS keys.

## Specification
### Mock CoinLookup

A `MockCoinLookup` struct MUST be provided that implements the `CoinLookup` trait:

```rust
use std::collections::HashMap;
use chia_protocol::{Bytes32, CoinState};

pub struct MockCoinLookup {
    coins: HashMap<Bytes32, CoinState>,
    chain_height: u64,
    chain_timestamp: u64,
}

impl MockCoinLookup {
    pub fn new() -> Self { /* ... */ }

    /// Register a coin state for lookup.
    pub fn add_coin_state(&mut self, coin_id: Bytes32, state: CoinState) { /* ... */ }

    /// Set the chain height returned by get_chain_height().
    pub fn set_chain_height(&mut self, height: u64) { /* ... */ }

    /// Set the chain timestamp returned by get_chain_timestamp().
    pub fn set_chain_timestamp(&mut self, timestamp: u64) { /* ... */ }
}

impl CoinLookup for MockCoinLookup {
    fn get_coin_state(&self, coin_id: &Bytes32) -> Option<CoinState> {
        self.coins.get(coin_id).cloned()
    }

    fn get_chain_height(&self) -> u64 {
        self.chain_height
    }

    fn get_chain_timestamp(&self) -> u64 {
        self.chain_timestamp
    }
}
```

### Mock BlockSigner

A `MockBlockSigner` struct MUST be provided that implements the `BlockSigner` trait:

```rust
use chia_bls::{SecretKey, Signature, sign};
use chia_protocol::Bytes32;

pub struct MockBlockSigner {
    secret_key: SecretKey,
}

impl MockBlockSigner {
    /// Create a mock signer with a deterministic test key.
    pub fn new() -> Self { /* ... */ }

    /// Get the public key corresponding to this signer.
    pub fn public_key(&self) -> PublicKey { /* ... */ }
}

impl BlockSigner for MockBlockSigner {
    fn sign_block(&self, block_hash: &Bytes32) -> Result<Signature, SignerError> {
        Ok(sign(&self.secret_key, block_hash.as_ref()))
    }
}
```

All test infrastructure SHOULD be gated behind `#[cfg(test)]` or placed in a dedicated `tests/` support module. If the infrastructure is intended for use by downstream crates, it MAY be placed behind a `test-utils` feature flag.

## Acceptance Criteria
- [ ] `MockCoinLookup` struct exists and implements `CoinLookup`
- [ ] `MockCoinLookup` allows adding arbitrary coin states via `add_coin_state`
- [ ] `MockCoinLookup` allows setting chain height and timestamp
- [ ] `MockBlockSigner` struct exists and implements `BlockSigner`
- [ ] `MockBlockSigner` produces valid BLS signatures
- [ ] `test_header()` helper returns a valid `L2BlockHeader`
- [ ] `test_header_at_height(n)` helper returns a header at the specified height
- [ ] `test_block()` helper returns a valid `L2Block`
- [ ] `test_spend_bundle()` helper returns a structurally valid `SpendBundle`
- [ ] All mocks and helpers compile and pass basic smoke tests

## Implementation Notes
- Primary file: `src/test_utils.rs` or `tests/common/mod.rs`
- Dependencies: `chia-protocol` (CoinState, SpendBundle, Bytes32), `chia-bls` (SecretKey, Signature, sign), `CoinLookup` and `BlockSigner` traits
- Edge cases: The `MockBlockSigner` should use a deterministic key derived from a fixed seed so tests are reproducible. `test_spend_bundle()` need not be consensus-valid but must be structurally complete (non-empty coin_spends list, valid aggregated_signature field).
