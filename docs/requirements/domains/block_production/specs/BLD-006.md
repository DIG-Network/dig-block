## Summary

The `build()` method must integrate with the `BlockSigner` trait to sign the assembled block header. Signing failures must be mapped to `BuilderError::SigningFailed`.

## Specification

The `BlockSigner` trait defines the signing interface:

```rust
pub trait BlockSigner {
    fn sign_block(&self, header_hash: &Bytes32) -> Result<Signature, SignerError>;
}
```

Within `build()`:

```rust
// Compute header hash after all fields are finalized
let header_hash = header.hash();

// Sign via the trait
let proposer_signature = signer.sign_block(&header_hash)
    .map_err(|e| BuilderError::SigningFailed(e))?;

// Set signature on the block (BLK-003: proposer_signature is on L2Block, not inside L2BlockHeader)
block.proposer_signature = proposer_signature;
```

- The `header_hash` is computed from the fully assembled header (including two-pass `block_size`; HSH-001 preimage does not include the BLS signature, which lives on [`L2Block`](../../block_types/specs/BLK-003.md) in this crate).
- The `SignerError` from the trait maps to `BuilderError::SigningFailed` (ERR-004 carries a `String`, populated from [`SignerError`](../../crate_structure/specs/STR-004.md) `Display` / `to_string()`).
- The resulting `Signature` is stored in `L2Block::proposer_signature`.

## Acceptance Criteria

- [x] `build()` calls `signer.sign_block()` with the computed header hash.
- [x] On signing success, `proposer_signature` is set on the returned `L2Block` (BLK-003 layout).
- [x] On signing failure, `build()` returns `Err(BuilderError::SigningFailed)`.
- [x] The `SignerError` diagnostic is preserved via `Display` inside `BuilderError::SigningFailed`’s `String` (ERR-004).
- [x] The `BlockSigner` trait is object-safe (usable as `&dyn BlockSigner`).

## Implementation Notes

- The `BlockSigner` trait is designed to be implementation-agnostic: it can wrap a local private key, an HSM, a remote signing service, or a test mock.
- The header hash must be computed after all other header fields are finalized (including `block_size` from the two-pass computation) but before the signature field is set.
- Object safety (`&dyn BlockSigner`) is required so that `build()` can accept any signer implementation.
