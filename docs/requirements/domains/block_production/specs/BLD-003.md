## Summary

`add_slash_proposal()` must enforce both a maximum count of slash proposals per block and a maximum payload size per individual proposal.

## Specification

```rust
impl BlockBuilder {
    pub fn add_slash_proposal(
        &mut self,
        payload: Vec<u8>,
    ) -> Result<(), BuilderError> {
        // Count limit
        if self.slash_proposal_payloads.len() >= MAX_SLASH_PROPOSALS_PER_BLOCK {
            return Err(BuilderError::TooManySlashProposals {
                max: MAX_SLASH_PROPOSALS_PER_BLOCK,
            });
        }

        // Payload size limit
        if payload.len() > MAX_SLASH_PROPOSAL_PAYLOAD_BYTES {
            return Err(BuilderError::SlashProposalTooLarge {
                size: payload.len(),
                max: MAX_SLASH_PROPOSAL_PAYLOAD_BYTES,
            });
        }

        self.slash_proposal_payloads.push(payload);
        Ok(())
    }
}
```

- The count check uses `>=` against `MAX_SLASH_PROPOSALS_PER_BLOCK` to prevent the addition that would exceed the limit.
- The payload size check uses strict greater-than: a payload exactly at the limit is accepted.

## Acceptance Criteria

- [ ] `add_slash_proposal()` returns `Ok` when count is below `MAX_SLASH_PROPOSALS_PER_BLOCK` and payload size is within limit.
- [ ] `add_slash_proposal()` returns `Err(TooManySlashProposals)` when adding would exceed `MAX_SLASH_PROPOSALS_PER_BLOCK`.
- [ ] `add_slash_proposal()` returns `Err(SlashProposalTooLarge)` when `payload.len() > MAX_SLASH_PROPOSAL_PAYLOAD_BYTES`.
- [ ] On rejection, the builder state is unchanged.
- [ ] On success, the payload is appended to `slash_proposal_payloads`.

## Implementation Notes

- The count check runs before the size check; if both limits are violated, the count error is returned.
- `MAX_SLASH_PROPOSALS_PER_BLOCK` and `MAX_SLASH_PROPOSAL_PAYLOAD_BYTES` are protocol constants.
