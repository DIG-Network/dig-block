## Summary

`BlockBuilder` must provide three optional setter methods for configuring L1 proof anchors, DFSP roots, and extension data. These fields are optional during block construction and may be omitted for pre-activation blocks.

## Specification

```rust
impl BlockBuilder {
    pub fn set_l1_proofs(
        &mut self,
        collateral: Bytes32,
        reserve: Bytes32,
        prev_finalizer: Bytes32,
        curr_finalizer: Bytes32,
        network_coin: Bytes32,
    ) {
        self.l1_collateral_coin_id = Some(collateral);
        self.l1_reserve_coin_id = Some(reserve);
        self.l1_prev_epoch_finalizer_coin_id = Some(prev_finalizer);
        self.l1_curr_epoch_finalizer_coin_id = Some(curr_finalizer);
        self.l1_network_coin_id = Some(network_coin);
    }

    pub fn set_dfsp_roots(
        &mut self,
        collateral_registry_root: Bytes32,
        cid_state_root: Bytes32,
        node_registry_root: Bytes32,
        namespace_update_root: Bytes32,
        dfsp_finalize_commitment_root: Bytes32,
    ) {
        self.collateral_registry_root = collateral_registry_root;
        self.cid_state_root = cid_state_root;
        self.node_registry_root = node_registry_root;
        self.namespace_update_root = namespace_update_root;
        self.dfsp_finalize_commitment_root = dfsp_finalize_commitment_root;
    }

    pub fn set_extension_data(&mut self, extension_data: Bytes32) {
        self.extension_data = extension_data;
    }
}
```

- `set_l1_proofs()` accepts five `Bytes32` values for the L1 proof anchors.
- `set_dfsp_roots()` accepts five `Bytes32` values for the DFSP Merkle roots.
- `set_extension_data()` accepts a single `Bytes32` for the extension data field.
- All setters are infallible (no error returns).
- If not called, the corresponding fields default to their zero/empty values.

## Acceptance Criteria

- [x] `set_l1_proofs()` stores all five L1 proof anchor values.
- [x] `set_dfsp_roots()` stores all five DFSP root values.
- [x] `set_extension_data()` stores the extension data value.
- [x] Calling a setter overwrites any previously set value.
- [x] Fields default to zero/empty if the corresponding setter is never called.

## Implementation Notes

- These setters exist because their fields are optional depending on the block version and L1 state.
- Pre-DFSP-activation blocks should not call `set_dfsp_roots()` (or pass `EMPTY_ROOT` values).
- The `build()` method (BLD-005) checks whether DFSP roots are required and rejects if they are missing.
- Rust `BlockBuilder` uses the same field names as [`L2BlockHeader`](../../block_types/specs/BLK-001.md) for L1/DFSP/extension (`l1_*_coin_id`, DFSP roots, `extension_data`) so `build()` can copy without a mapping layer.
