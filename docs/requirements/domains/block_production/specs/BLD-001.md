## Summary

The `BlockBuilder` struct must contain all fields necessary to incrementally construct an L2 block, and its `new()` constructor must initialize the builder with empty collections and zero totals.

## Specification

```rust
pub struct BlockBuilder {
    pub height: u64,
    pub epoch: u64,
    pub parent_hash: Bytes32,
    pub l1_height: u32,
    pub l1_hash: Bytes32,
    pub proposer_index: u32,
    pub spend_bundles: Vec<SpendBundle>,
    pub slash_proposal_payloads: Vec<Vec<u8>>,
    pub total_cost: Cost,
    pub total_fees: u64,
    pub additions: Vec<Coin>,
    pub removals: Vec<Bytes32>, // NORMATIVE `CoinId` — 32-byte coin id (see BLK-004 / L2Block::all_removals)
}

impl BlockBuilder {
    pub fn new(
        height: u64,
        epoch: u64,
        parent_hash: Bytes32,
        l1_height: u32,
        l1_hash: Bytes32,
        proposer_index: u32,
    ) -> Self {
        Self {
            height,
            epoch,
            parent_hash,
            l1_height,
            l1_hash,
            proposer_index,
            spend_bundles: Vec::new(),
            slash_proposal_payloads: Vec::new(),
            total_cost: 0,
            total_fees: 0,
            additions: Vec::new(),
            removals: Vec::new(),
        }
    }
}
```

The constructor accepts the immutable block identity fields (height, epoch, parent_hash, l1_height, l1_hash, proposer_index) and initializes all mutable accumulation fields to their zero/empty defaults.

## Acceptance Criteria

- [x] `BlockBuilder` struct contains all 12 specified fields with correct types.
- [x] `new()` accepts height, epoch, parent_hash, l1_height, l1_hash, and proposer_index.
- [x] `new()` sets `spend_bundles` to an empty `Vec`.
- [x] `new()` sets `slash_proposal_payloads` to an empty `Vec`.
- [x] `new()` sets `total_cost` to 0.
- [x] `new()` sets `total_fees` to 0.
- [x] `new()` sets `additions` to an empty `Vec`.
- [x] `new()` sets `removals` to an empty `Vec`.

## Implementation Notes

- The `Cost` type alias is typically `u64` representing CLVM execution cost units.
- `Bytes32` is a 32-byte fixed-size array type used throughout the protocol.
- `CoinId` in prose is the SHA-256 hash identifying a coin; the pinned `chia-protocol` crate does not define a `CoinId`
  newtype, so the Rust field is `Vec<Bytes32>` (same element type as `L2Block::all_removals()` in the crate).
