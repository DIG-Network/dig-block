## Summary

`add_spend_bundle()` must enforce cost and size budgets before accepting a spend bundle, then extract its additions and removals, update running totals, and append the bundle.

## Specification

```rust
impl BlockBuilder {
    pub fn add_spend_bundle(
        &mut self,
        bundle: SpendBundle,
        cost: Cost,
        fee: u64,
    ) -> Result<(), BuilderError> {
        // Budget check: cost
        if self.total_cost + cost > MAX_COST_PER_BLOCK {
            return Err(BuilderError::CostBudgetExceeded {
                current: self.total_cost,
                additional: cost,
                max: MAX_COST_PER_BLOCK,
            });
        }

        // Budget check: estimated size
        let estimated_size = self.estimate_size_with(&bundle);
        if estimated_size > MAX_BLOCK_SIZE {
            return Err(BuilderError::SizeBudgetExceeded {
                estimated: estimated_size,
                max: MAX_BLOCK_SIZE,
            });
        }

        // Extract additions and removals
        for coin_spend in &bundle.coin_spends {
            self.additions.extend(coin_spend.additions());
            self.removals.push(coin_spend.coin.coin_id());
        }

        // Update totals
        self.total_cost += cost;
        self.total_fees += fee;

        // Append bundle
        self.spend_bundles.push(bundle);

        Ok(())
    }

    pub fn remaining_cost(&self) -> Cost {
        MAX_COST_PER_BLOCK - self.total_cost
    }

    pub fn spend_bundle_count(&self) -> usize {
        self.spend_bundles.len()
    }
}
```

- The cost check prevents exceeding `MAX_COST_PER_BLOCK`.
- The size check estimates the block size if this bundle were included and prevents exceeding `MAX_BLOCK_SIZE`.
- Both checks occur before any mutation so that a rejected bundle leaves the builder unchanged.
- `remaining_cost()` returns the remaining cost budget.

## Acceptance Criteria

- [ ] `add_spend_bundle()` returns `Err(CostBudgetExceeded)` when `total_cost + cost > MAX_COST_PER_BLOCK`.
- [ ] `add_spend_bundle()` returns `Err(SizeBudgetExceeded)` when estimated size exceeds `MAX_BLOCK_SIZE`.
- [ ] On rejection, the builder state is unchanged (no partial mutations).
- [ ] On success, additions are extracted from the bundle's coin spends.
- [ ] On success, removals (coin IDs) are extracted from the bundle's coin spends.
- [ ] On success, `total_cost` is incremented by `cost`.
- [ ] On success, `total_fees` is incremented by `fee`.
- [ ] On success, the bundle is appended to `spend_bundles`.
- [ ] `remaining_cost()` returns `MAX_COST_PER_BLOCK - total_cost`.
- [ ] `spend_bundle_count()` returns the number of SpendBundles added so far.

## Implementation Notes

- The size estimation method (`estimate_size_with`) should account for the serialized size of the bundle plus existing block contents.
- The builder must remain in a consistent state after a rejected `add_spend_bundle` call -- no fields should be partially updated.
- Multiple bundles can be added sequentially, each incrementing the running totals.
