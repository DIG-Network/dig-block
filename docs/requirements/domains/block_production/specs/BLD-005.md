## Summary

`build()` orchestrates the final assembly of an `L2Block` from the accumulated builder state. It computes all derived fields (Merkle roots, filter hash, version, timestamp, size), signs the header, and returns the complete block.

## Specification

```rust
impl BlockBuilder {
    pub fn build(
        self,
        state_root: Bytes32,
        receipts_root: Bytes32,
        signer: &dyn BlockSigner,
    ) -> Result<L2Block, BuilderError> {
        // Step 1: Compute spends_root
        let spend_hashes: Vec<Bytes32> = self.spend_bundles.iter()
            .map(|sb| sb.hash())
            .collect();
        let spends_root = MerkleTree::new(&spend_hashes).root();

        // Step 2: Compute additions_root (grouped by puzzle_hash)
        let additions_root = compute_merkle_set_root(
            &group_by_puzzle_hash(&self.additions)
        );

        // Step 3: Compute removals_root
        let removals_root = compute_merkle_set_root(&self.removals);

        // Step 4: Compute filter_hash (BIP158)
        let filter = build_gcs_filter(&self.additions, &self.removals);
        let filter_hash = filter.hash();

        // Step 5: Compute slash_proposals_root
        let slash_proposals_root = compute_slash_proposals_root(
            &self.slash_proposal_payloads
        );

        // Step 6: Count all items
        let spend_bundle_count = self.spend_bundles.len() as u32;
        let additions_count = self.additions.len() as u32;
        let removals_count = self.removals.len() as u32;
        let slash_proposal_count = self.slash_proposal_payloads.len() as u32;

        // Step 7: Auto-detect version
        let version = if DFSP_ACTIVATION_HEIGHT == u64::MAX
            || self.height < DFSP_ACTIVATION_HEIGHT
        {
            VERSION_V1
        } else {
            VERSION_V2
        };

        // Step 7b: Validate DFSP roots if required
        if version == VERSION_V2 && !self.dfsp_roots_set {
            return Err(BuilderError::MissingDfspRoots);
        }

        // Step 8: Set timestamp
        let timestamp = current_unix_timestamp();

        // Step 9: Compute block_size (two-pass)
        // First pass: assemble header with block_size = 0
        // Measure serialized size
        // Second pass: update block_size field
        let mut header = L2BlockHeader { /* ... all fields ... */ block_size: 0, .. };
        let block = L2Block { header, spend_bundles, slash_proposal_payloads };
        let measured_size = block.serialized_size();
        block.header.block_size = measured_size as u64;

        // Step 10: Sign header
        let header_hash = block.header.hash();
        let signature = signer.sign_block(&header_hash)
            .map_err(|e| BuilderError::SigningFailed(e))?;
        block.header.proposer_signature = signature;

        // Step 11: Return complete block
        Ok(block)
    }
}
```

## Acceptance Criteria

- [x] `build()` computes `spends_root` via `MerkleTree` over spend bundle hashes.
- [x] `build()` computes `additions_root` via `compute_merkle_set_root` grouped by `puzzle_hash`.
- [x] `build()` computes `removals_root` via `compute_merkle_set_root`.
- [x] `build()` computes `filter_hash` via BIP158 GCS filter.
- [x] `build()` computes `slash_proposals_root` from payloads.
- [x] `build()` correctly counts spend bundles, additions, removals, and slash proposals.
- [x] `build()` auto-detects version as V1 when pre-activation or DFSP disabled, V2 otherwise.
- [x] `build()` returns `Err(MissingDfspRoots)` when V2 is required but DFSP roots were not set.
- [x] `build()` sets timestamp to current wall-clock time.
- [x] `build()` uses a two-pass approach for `block_size`: assemble with 0, measure, then update.
- [x] `build()` calls `signer.sign_block()` and sets `proposer_signature`.
- [x] `build()` returns a complete `L2Block`.

## Implementation Notes

- The two-pass size computation is necessary because the `block_size` field is part of the header, which is part of the serialized block. Assembling with 0, measuring, and updating avoids a circular dependency.
- The `state_root` and `receipts_root` are computed externally (by the state machine) and passed into `build()`.
- This method consumes `self` (takes ownership) because the builder is not reusable after building.
