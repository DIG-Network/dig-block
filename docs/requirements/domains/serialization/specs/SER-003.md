# SER-003: Genesis Block Construction

- **Normative:** [NORMATIVE.md](../NORMATIVE.md#ser-003-genesis-block-construction)
- **Verification:** [VERIFICATION.md](../VERIFICATION.md)
- **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
- **Spec:** SPEC Section 8.3

## Summary

[`L2BlockHeader::genesis`](../../../../../src/types/header.rs) constructs the genesis L2 header with deterministic bootstrap fields: no parent block (`parent_hash = network_id`), empty Merkle / DFSP roots, zero counts and costs, optional L1 proof slots unset, and a wall-clock Unix `timestamp`. **`version`** follows the same **BLK-007** height + DFSP activation rule as [`L2BlockHeader::new`](../../../../../src/types/header.rs) (not `CARGO_PKG_VERSION` — older pseudocode below is illustrative only).

## Specification

**Authoritative implementation:** [`L2BlockHeader::genesis`](../../../../../src/types/header.rs) delegates to [`with_l1_anchors`](../../../../../src/types/header.rs) so every root and optional field stays aligned with the full struct (spends/additions/removals roots, slash summary, five DFSP roots, five L1 `Option` anchors).

Illustrative shape (field names in this repo; not copy-paste complete):

```rust
pub fn genesis(network_id: Bytes32, l1_height: u32, l1_hash: Bytes32) -> Self {
    let timestamp = SystemTime::now()
        .duration_since(UNIX_EPOCH)
        .expect("system clock before UNIX epoch; genesis header requires wall-clock time")
        .as_secs();
    Self::with_l1_anchors(
        /* height */ 0,
        /* epoch */ 0,
        /* parent_hash */ network_id,
        /* state, spends, additions, removals, receipts */ EMPTY_ROOT, EMPTY_ROOT, EMPTY_ROOT, EMPTY_ROOT, EMPTY_ROOT,
        l1_height,
        l1_hash,
        timestamp,
        /* proposer, spend_bundle_count, cost, fees, counts, block_size, filter */ 0, 0, 0, 0, 0, 0, 0, EMPTY_ROOT,
        /* extension_data */ ZERO_HASH,
        /* L1 proof coin ids */ None, None, None, None, None,
        /* slash_proposal_count */ 0,
        /* slash_proposals_root + DFSP roots */ EMPTY_ROOT, EMPTY_ROOT, EMPTY_ROOT, EMPTY_ROOT, EMPTY_ROOT, EMPTY_ROOT,
    )
}
// `version` is set inside `with_l1_anchors` via `protocol_version_for_height(height)` (BLK-007).
```

## Acceptance Criteria

- [x] `height` is set to 0
- [x] `epoch` is set to 0
- [x] `parent_hash` is set to `network_id`
- [x] `state_root`, `spends_root`, `additions_root`, `removals_root`, `receipts_root`, `filter_hash`, `slash_proposals_root`, and all DFSP roots are set to `EMPTY_ROOT`
- [x] `extension_data` is set to `ZERO_HASH`
- [x] All counts and costs (`spend_bundle_count`, `additions_count`, `removals_count`, `slash_proposal_count`, `total_cost`, `total_fees`, `block_size`, `proposer_index`) are set to 0
- [x] All L1 proof coin id fields are set to `None`
- [x] `timestamp` is set to current wall-clock time (Unix seconds)
- [x] `version` equals `L2BlockHeader::protocol_version_for_height(0)` (BLK-007; with default `DFSP_ACTIVATION_HEIGHT` this is `VERSION_V1`)

## Implementation Notes

- `EMPTY_ROOT` / `ZERO_HASH` are crate constants ([BLK-005](docs/requirements/domains/block_types/specs/BLK-005.md)); genesis never invents ad-hoc roots.
- `parent_hash = network_id` prevents cross-network replay at height 0.
- Wall-clock `timestamp` implies different genesis construction times yield different header hashes (expected).
- **Version:** must stay tied to [`protocol_version_for_height`](../../../../../src/types/header.rs) so SVL-001 and genesis stay one rule.
