# SER-002: to_bytes and from_bytes Conventions

- **Normative:** [NORMATIVE.md](../NORMATIVE.md#ser-002-to_bytes-and-from_bytes-conventions)
- **Verification:** [VERIFICATION.md](../VERIFICATION.md)
- **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
- **Spec:** SPEC Section 8.2

## Summary

`to_bytes()` must be infallible (panics on failure, which should never occur). `from_bytes()` must be fallible, returning typed errors on invalid input.

## Specification

```rust
impl L2BlockHeader {
    /// Serialize to bytes. Infallible - panics on failure (should never happen).
    pub fn to_bytes(&self) -> Vec<u8> {
        bincode::serialize(self).expect("L2BlockHeader serialization should never fail")
    }

    /// Deserialize from bytes. Returns error on invalid data.
    pub fn from_bytes(bytes: &[u8]) -> Result<Self, BlockError> {
        bincode::deserialize(bytes)
            .map_err(|e| BlockError::InvalidData(e.to_string()))
    }
}

impl L2Block {
    pub fn to_bytes(&self) -> Vec<u8> {
        bincode::serialize(self).expect("L2Block serialization should never fail")
    }

    pub fn from_bytes(bytes: &[u8]) -> Result<Self, BlockError> {
        bincode::deserialize(bytes)
            .map_err(|e| BlockError::InvalidData(e.to_string()))
    }
}

impl Checkpoint {
    pub fn to_bytes(&self) -> Vec<u8> {
        bincode::serialize(self).expect("Checkpoint serialization should never fail")
    }

    pub fn from_bytes(bytes: &[u8]) -> Result<Self, CheckpointError> {
        bincode::deserialize(bytes)
            .map_err(|e| CheckpointError::InvalidData(e.to_string()))
    }
}

// Same pattern for AttestedBlock, CheckpointSubmission
```

## Acceptance Criteria

- [ ] `to_bytes()` returns `Vec<u8>` (not `Result`) - infallible
- [ ] `to_bytes()` panics via `expect()` on serialization failure
- [ ] `from_bytes()` returns `Result<Self, BlockError>` for block types
- [ ] `from_bytes()` returns `Result<Self, CheckpointError>` for checkpoint types
- [ ] Invalid bytes produce `BlockError::InvalidData` or `CheckpointError::InvalidData`
- [ ] Truncated bytes produce appropriate error
- [ ] Empty bytes produce appropriate error

## Implementation Notes

- The infallible `to_bytes()` convention is safe because bincode serialization of well-formed Rust structs with serde should never fail. The `expect()` message documents this invariant.
- `from_bytes()` must handle all categories of invalid input: truncated data, corrupted data, extra trailing bytes (depending on bincode configuration), and type mismatches.
