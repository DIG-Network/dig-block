# SER-001: Bincode Serialization for All Types

- **Normative:** [NORMATIVE.md](../NORMATIVE.md#ser-001-bincode-serialization-for-all-types)
- **Verification:** [VERIFICATION.md](../VERIFICATION.md)
- **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
- **Spec:** SPEC Section 8.1

## Summary

All block types must use bincode for serialization. Bincode produces compact binary output with no schema overhead, making it efficient for network transmission and storage.

## Specification

```rust
use bincode::{serialize, deserialize};

// All block types derive or implement serde Serialize/Deserialize
// and use bincode as the serialization format.

#[derive(Serialize, Deserialize)]
pub struct L2BlockHeader { /* ... */ }

#[derive(Serialize, Deserialize)]
pub struct L2Block { /* ... */ }

#[derive(Serialize, Deserialize)]
pub struct AttestedBlock { /* ... */ }

#[derive(Serialize, Deserialize)]
pub struct Checkpoint { /* ... */ }

#[derive(Serialize, Deserialize)]
pub struct CheckpointSubmission { /* ... */ }

// Bincode is used exclusively for wire/storage format
impl L2BlockHeader {
    pub fn to_bytes(&self) -> Vec<u8> {
        bincode::serialize(self).expect("serialization should never fail")
    }

    pub fn from_bytes(bytes: &[u8]) -> Result<Self, BlockError> {
        bincode::deserialize(bytes).map_err(|e| BlockError::InvalidData(e.to_string()))
    }
}

// Same pattern for L2Block, AttestedBlock, Checkpoint, CheckpointSubmission
```

## Acceptance Criteria

- [ ] `L2BlockHeader` uses bincode for serialization
- [ ] `L2Block` uses bincode for serialization
- [ ] `AttestedBlock` uses bincode for serialization
- [ ] `Checkpoint` uses bincode for serialization
- [ ] `CheckpointSubmission` uses bincode for serialization
- [ ] `SignerBitmap` derives `Serialize` and `Deserialize`
- [ ] `Receipt` and `ReceiptList` derive `Serialize` and `Deserialize`
- [ ] `BlockStatus` and `CheckpointStatus` derive `Serialize` and `Deserialize`
- [ ] `ExecutionResult` and `PendingAssertion` derive `Serialize` and `Deserialize`
- [ ] No other serialization format (JSON, MessagePack, etc.) is used for wire/storage
- [ ] All types derive or implement `serde::Serialize` and `serde::Deserialize`

## Implementation Notes

- Bincode's default configuration uses little-endian, variable-length integer encoding. If fixed configuration is needed for cross-platform compatibility, use `bincode::options()` to set explicit options.
- JSON serialization may be provided separately for debugging/logging but MUST NOT be used for consensus-critical serialization.
