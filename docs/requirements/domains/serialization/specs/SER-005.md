# SER-005: Serialization Round-Trip Integrity

- **Normative:** [NORMATIVE.md](../NORMATIVE.md#ser-005-serialization-round-trip-integrity)
- **Verification:** [VERIFICATION.md](../VERIFICATION.md)
- **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
- **Spec:** SPEC Section 12.1 (Serialization round-trips), 12.5

## Summary

For all types, `from_bytes(to_bytes(x))` MUST equal `x`. No data loss is permitted through serialization round-trips. This property must be verified using property-based testing.

## Specification

```rust
#[cfg(test)]
mod tests {
    use proptest::prelude::*;

    // Arbitrary implementations for all block types
    impl Arbitrary for L2BlockHeader {
        type Parameters = ();
        type Strategy = BoxedStrategy<Self>;

        fn arbitrary_with(_: ()) -> Self::Strategy {
            (
                any::<u32>(),       // version
                any::<u64>(),       // height
                any::<u64>(),       // epoch
                any::<[u8; 32]>(),  // parent_hash
                any::<[u8; 32]>(),  // state_root
                // ... all fields
            )
            .prop_map(|(version, height, epoch, parent_hash, state_root, ...)| {
                L2BlockHeader {
                    version,
                    height,
                    epoch,
                    parent_hash: parent_hash.into(),
                    state_root: state_root.into(),
                    // ... all fields
                }
            })
            .boxed()
        }
    }

    proptest! {
        #[test]
        fn roundtrip_l2_block_header(header in any::<L2BlockHeader>()) {
            let bytes = header.to_bytes();
            let decoded = L2BlockHeader::from_bytes(&bytes).unwrap();
            prop_assert_eq!(header, decoded);
        }

        #[test]
        fn roundtrip_l2_block(block in any::<L2Block>()) {
            let bytes = block.to_bytes();
            let decoded = L2Block::from_bytes(&bytes).unwrap();
            prop_assert_eq!(block, decoded);
        }

        #[test]
        fn roundtrip_attested_block(block in any::<AttestedBlock>()) {
            let bytes = block.to_bytes();
            let decoded = AttestedBlock::from_bytes(&bytes).unwrap();
            prop_assert_eq!(block, decoded);
        }

        #[test]
        fn roundtrip_checkpoint(cp in any::<Checkpoint>()) {
            let bytes = cp.to_bytes();
            let decoded = Checkpoint::from_bytes(&bytes).unwrap();
            prop_assert_eq!(cp, decoded);
        }

        #[test]
        fn roundtrip_checkpoint_submission(sub in any::<CheckpointSubmission>()) {
            let bytes = sub.to_bytes();
            let decoded = CheckpointSubmission::from_bytes(&bytes).unwrap();
            prop_assert_eq!(sub, decoded);
        }
    }
}
```

## Acceptance Criteria

- [ ] `from_bytes(to_bytes(x)) == x` for `L2BlockHeader`
- [ ] `from_bytes(to_bytes(x)) == x` for `L2Block`
- [ ] `from_bytes(to_bytes(x)) == x` for `AttestedBlock`
- [ ] `from_bytes(to_bytes(x)) == x` for `Checkpoint`
- [ ] `from_bytes(to_bytes(x)) == x` for `CheckpointSubmission`
- [ ] Property tests use proptest or equivalent framework
- [ ] Edge cases covered: empty collections, maximum values, zero values
- [ ] All types implement `PartialEq` for comparison

## Implementation Notes

- Property-based testing generates thousands of random inputs, providing much stronger guarantees than hand-written test cases. Use `proptest` with at least the default number of cases (256).
- All types must derive `PartialEq` (and ideally `Debug`) for property test assertions.
- If `Arbitrary` is complex to implement for nested types, consider using `proptest-derive` or manual strategy composition.
- Edge cases to pay special attention to: `Bytes32` with all zeros, all ones; `Vec` fields that are empty; `Option` fields that are `None` and `Some`; maximum u64 values.
