## Summary

Defines the `AttestedBlock` struct and its constructor. The struct wraps an L2 block with attestation metadata including a signer bitmap, aggregate signature, receipts, and status. The constructor initializes the block in a `Pending` state with an empty signer bitmap.

## Specification

`AttestedBlock` contains the following fields:

| Field                 | Type            | Description                            |
|-----------------------|-----------------|----------------------------------------|
| `block`               | `L2Block`       | The L2 block being attested            |
| `signer_bitmap`       | `SignerBitmap`  | Tracks which validators have signed    |
| `aggregate_signature` | `Signature`     | Aggregated BLS signature               |
| `receipts`            | `ReceiptList`   | Transaction receipts for the block     |
| `status`              | `BlockStatus`   | Current attestation status             |

### Constructor

```rust
impl AttestedBlock {
    pub fn new(block: L2Block, validator_count: u32, receipts: ReceiptList) -> Self {
        Self {
            aggregate_signature: block.proposer_signature(),
            signer_bitmap: SignerBitmap::new(validator_count),
            status: BlockStatus::Pending,
            block,
            receipts,
        }
    }
}
```

- The signer bitmap MUST be initialized as empty (no validators signed).
- The status MUST be initialized to `Pending`.
- The `aggregate_signature` MUST be set to the proposer's signature from the block.

## Acceptance Criteria

- [ ] `AttestedBlock` struct contains all five required fields with correct types.
- [ ] `new()` initializes `signer_bitmap` as empty.
- [ ] `new()` initializes `status` to `BlockStatus::Pending`.
- [ ] `new()` sets `aggregate_signature` to the proposer's signature.
- [ ] `new()` stores the provided `block` and `receipts`.

## Implementation Notes

- The proposer's signature serves as the initial aggregate signature before any validator attestations are added.
- The signer bitmap size is determined by `validator_count`, which may differ per epoch.
