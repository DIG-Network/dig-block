## Summary

Defines the `merge` and `signer_indices` methods on `SignerBitmap`. Merge combines two bitmaps using bitwise OR. `signer_indices` returns an ordered list of all validator indices that have signed.

## Specification

### merge

```rust
impl SignerBitmap {
    pub fn merge(&mut self, other: &SignerBitmap) -> Result<(), Error> {
        if self.validator_count != other.validator_count {
            return Err(Error::ValidatorCountMismatch);
        }
        for (a, b) in self.bits.iter_mut().zip(other.bits.iter()) {
            *a |= *b;
        }
        Ok(())
    }
}
```

Performs a bitwise OR of the two bitmaps, combining signer sets. MUST reject bitmaps with mismatched `validator_count`.

### signer_indices

```rust
impl SignerBitmap {
    pub fn signer_indices(&self) -> Vec<u32> {
        let mut indices = Vec::new();
        for i in 0..self.validator_count {
            if self.has_signed(i) {
                indices.push(i);
            }
        }
        indices
    }
}
```

Returns an ordered list of all validator indices that have signed.

## Acceptance Criteria

- [ ] `merge(other)` performs bitwise OR across all bytes.
- [ ] `merge(other)` returns `Err` when `validator_count` values differ.
- [ ] `merge(other)` succeeds when `validator_count` values match.
- [ ] After merge, signers from both bitmaps are present.
- [ ] `signer_indices()` returns indices in ascending order.
- [ ] `signer_indices()` returns an empty vector for an empty bitmap.
- [ ] `signer_indices()` returns correct indices after `set_signed` calls.

## Implementation Notes

- Merge is commutative: `a.merge(b)` produces the same signer set as `b.merge(a)`.
- `signer_indices` iterates from index 0 to `validator_count - 1`, naturally producing sorted output.
- Merge is useful for aggregating attestations received from different peers.
