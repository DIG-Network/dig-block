## Summary

Defines the `BlockStatus` enum representing the lifecycle states of an attested block, along with query methods for finality and canonicality.

## Specification

### Enum Variants

```rust
pub enum BlockStatus {
    Pending,
    Validated,
    SoftFinalized,
    HardFinalized,
    Orphaned,
    Rejected,
}
```

### is_finalized

```rust
impl BlockStatus {
    pub fn is_finalized(&self) -> bool {
        matches!(self, BlockStatus::SoftFinalized | BlockStatus::HardFinalized)
    }
}
```

Returns `true` for `SoftFinalized` and `HardFinalized`; `false` for all other variants.

### is_canonical

```rust
impl BlockStatus {
    pub fn is_canonical(&self) -> bool {
        !matches!(self, BlockStatus::Orphaned | BlockStatus::Rejected)
    }
}
```

Returns `false` for `Orphaned` and `Rejected`; `true` for all other variants.

## Acceptance Criteria

- [ ] `BlockStatus` defines exactly six variants: Pending, Validated, SoftFinalized, HardFinalized, Orphaned, Rejected.
- [ ] `is_finalized()` returns `true` only for `SoftFinalized` and `HardFinalized`.
- [ ] `is_finalized()` returns `false` for Pending, Validated, Orphaned, and Rejected.
- [ ] `is_canonical()` returns `false` only for `Orphaned` and `Rejected`.
- [ ] `is_canonical()` returns `true` for Pending, Validated, SoftFinalized, and HardFinalized.

## Implementation Notes

- `Pending` is the initial state set by the `AttestedBlock` constructor.
- A block that is `SoftFinalized` has met the signing threshold but has not yet been confirmed on L1.
- `HardFinalized` indicates L1 confirmation via checkpoint.
- `Orphaned` blocks are superseded by a competing fork; `Rejected` blocks failed validation.
