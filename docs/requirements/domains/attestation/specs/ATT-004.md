## Summary

Defines the `SignerBitmap` struct and its core methods for tracking which validators have signed a block. The bitmap uses a compact bit-vector representation with a maximum of 65536 validators.

## Specification

### Struct

```rust
pub const MAX_VALIDATORS: u32 = 65536;

pub struct SignerBitmap {
    bits: Vec<u8>,
    validator_count: u32,
}
```

### new

```rust
impl SignerBitmap {
    pub fn new(validator_count: u32) -> Self {
        let byte_count = (validator_count as usize + 7) / 8;
        Self {
            bits: vec![0u8; byte_count],
            validator_count,
        }
    }
}
```

Creates a new empty bitmap sized for `validator_count` validators.

### from_bytes

```rust
impl SignerBitmap {
    pub fn from_bytes(bytes: &[u8], validator_count: u32) -> Self {
        Self {
            bits: bytes.to_vec(),
            validator_count,
        }
    }
}
```

Creates a bitmap from raw bytes and a validator count.

### has_signed

```rust
impl SignerBitmap {
    pub fn has_signed(&self, index: u32) -> bool {
        let byte_index = (index / 8) as usize;
        let bit_index = index % 8;
        self.bits[byte_index] & (1 << bit_index) != 0
    }
}
```

Returns whether the validator at `index` has signed.

### set_signed

```rust
impl SignerBitmap {
    pub fn set_signed(&mut self, index: u32) -> Result<(), Error> {
        if index >= self.validator_count {
            return Err(Error::IndexOutOfBounds);
        }
        let byte_index = (index / 8) as usize;
        let bit_index = index % 8;
        self.bits[byte_index] |= 1 << bit_index;
        Ok(())
    }
}
```

Marks the validator at `index` as signed. Returns an error if the index is out of bounds.

### signer_count

```rust
impl SignerBitmap {
    pub fn signer_count(&self) -> u32 {
        self.bits.iter().map(|b| b.count_ones()).sum()
    }
}
```

Returns the total number of validators that have signed.

### signing_percentage

```rust
impl SignerBitmap {
    pub fn signing_percentage(&self) -> u64 {
        if self.validator_count == 0 {
            return 0;
        }
        (self.signer_count() as u64 * 100) / self.validator_count as u64
    }
}
```

Returns the signing percentage as an integer in the range 0-100.

### has_threshold

```rust
impl SignerBitmap {
    pub fn has_threshold(&self, threshold_pct: u64) -> bool {
        self.signing_percentage() >= threshold_pct
    }
}
```

Returns whether the signing percentage meets or exceeds the threshold.

### as_bytes

```rust
impl SignerBitmap {
    pub fn as_bytes(&self) -> &[u8] {
        &self.bits
    }
}
```

Returns the raw bitmap bytes.

## Acceptance Criteria

- [ ] `SignerBitmap` contains `bits: Vec<u8>` and `validator_count: u32`.
- [ ] `MAX_VALIDATORS` is defined as `65536`.
- [ ] `new(validator_count)` creates a zero-filled bitmap of correct byte length.
- [ ] `from_bytes(bytes, validator_count)` reconstructs a bitmap from raw bytes.
- [ ] `has_signed(index)` returns correct bit state for any valid index.
- [ ] `set_signed(index)` sets the correct bit and returns `Ok(())`.
- [ ] `set_signed(index)` returns `Err` when index >= validator_count.
- [ ] `signer_count()` returns the correct popcount across all bytes.
- [ ] `signing_percentage()` returns 0-100 and handles validator_count == 0.
- [ ] `has_threshold(threshold_pct)` returns correct boolean comparison.
- [ ] `as_bytes()` returns a reference to the internal bits vector.

## Implementation Notes

- Byte length is `ceil(validator_count / 8)`.
- Bit ordering is LSB-first within each byte (bit 0 of byte 0 = validator index 0).
- `signer_count` uses `count_ones()` (popcount) for efficiency.
- When `validator_count` is 0, `signing_percentage` must return 0 to avoid division by zero.
